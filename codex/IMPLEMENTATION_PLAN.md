# 经纬项目实施计划

> 用途：AI 辅助开发的分阶段落地计划。每个阶段要继续拆成 `TASKS.md` 中的小任务。
> 更新时间：2026-04-30。

## 规划原则

- 先打基础，再做复杂业务。
- 每次交付可运行、可验证的业务切片。
- 库存、状态机、Outbox、审计属于基础设施，不是后期优化。
- 优先做 P0，但支撑后续 P0 的公共能力要提前做。
- P1 功能要有清晰边界，不能混进 P0 任务里偷偷扩大范围。
- 每个阶段结束都要能构建、能测试，并更新 `codex/` 记忆文档。

## 阶段 0：项目基础骨架

目标：建立可运行的后端和前端基础。

后端交付：

- Java 17 + Spring Boot 3 项目。
- 根包名 `com.jingwei`。
- 模块包占位：`common`、`master`、`order`、`procurement`、`inventory`、`warehouse`、`approval`、`notification`、`system`。
- PostgreSQL 连接与 migration 工具。
- Redis 连接。
- 统一 API 响应结构。
- 统一业务异常和参数校验异常处理。
- 审计字段：`created_by`、`created_at`、`updated_by`、`updated_at`。
- 软删除字段：`deleted`。
- 乐观锁字段：`version`。
- 认证基础。
- JWT 签发和校验。
- BCrypt 密码保存。
- 用户、角色、权限基础模型。
- 操作日志骨架。
- 系统配置骨架。
- 测试框架。

前端交付：

- React 18 + Ant Design Pro 项目。
- 登录页。
- 主布局和菜单。
- API 客户端，支持 JWT。
- 权限路由/菜单占位。
- 通用列表、详情、表单模式。
- 扫码输入辅助占位。

验收：

- 后端可启动。
- 前端可启动。
- 空数据库 migration 可运行。
- 基础测试命令可运行。

## 阶段 1：共享领域基础设施

目标：实现后续业务模块依赖的公共能力。

交付：

- 轻量状态机：
  - 转移表
  - 条件策略
  - 动作策略
  - 监听器
  - 当前可用操作查询
  - 单元测试
- 事务性 Outbox：
  - `domain_event_outbox`
  - 领域事件模型
  - 事务内写 Outbox
  - Relay 投递 Spring ApplicationEvent
  - 失败重试
  - 消费幂等
- 幂等：
  - `X-Idempotency-Key`
  - Redis SETNX
  - 写接口防重复提交
- 编码规则引擎：
  - 编码规则表
  - 规则段渲染
  - 数据库行锁递增流水号
  - 预览接口
- 审计和操作日志。
- 系统配置默认值。

验收：

- 状态机测试覆盖合法转移、非法转移、监听器、条件失败。
- 编码并发测试不重复。
- Outbox 与业务事务同提交同回滚。
- 幂等测试证明重复 key 不重复执行。

## 阶段 2：基础数据 MVP

目标：为订单、采购、库存、仓库提供主数据。

范围：

- 物料分类树。
- 面料、辅料、包材物料主数据。
- 属性定义和 JSONB 扩展属性。
- 季节、波段。
- 尺码组、尺码。
- SPU。
- 颜色款。
- SKU 自动生成。
- 供应商。
- 客户。
- 仓库。
- 库位。
- 编码规则管理。

关键行为：

- 成品不进 `md_material`。
- 创建 SPU 必须选择颜色和尺码组。
- SKU 编码由款式编码、颜色编码、尺码编码拼接。
- SPU 生成 SKU 后不能更换尺码组。
- 被引用数据不能物理删除。
- 停用数据不能被新单据选择，但已有引用有效。

验收：

- 创建尺码组、SPU、颜色后自动生成 SKU。
- 创建不同类型物料时显示并保存不同扩展属性。
- 创建仓库和批量库位。
- 编码规则能安全生成编号。
- 列表支持分页和筛选。

## 阶段 3：系统管理 MVP

目标：让不同角色可以安全使用系统。

范围：

- 用户管理。
- 角色管理。
- 菜单权限。
- 按钮权限。
- 按仓库/部门的数据权限。
- 密码策略基础。
- 单用户单会话。
- 操作日志查询。
- 审批配置。
- 通知偏好。

验收：

- 预置角色存在。
- 权限控制菜单和按钮。
- 数据权限由后端查询层追加，不依赖前端。
- 操作日志只读。

## 阶段 4：库存核心 MVP

目标：先实现最关键的数据一致性域，再让订单、采购、仓库依赖它。

范围：

- `inventory_sku`
- `inventory_material`
- `inventory_in_transit`
- `inventory_operation`
- `inventory_allocation`
- 统一库存变更服务。
- 操作类型：
  - `INBOUND_PURCHASE`
  - `INBOUND_PRODUCTION`
  - `INBOUND_RETURN`
  - `QC_PASS`
  - `QC_FAIL`
  - `ALLOCATE`
  - `RELEASE`
  - `OUTBOUND_SALES`
  - `OUTBOUND_MATERIAL`
  - `ADJUST_GAIN`
  - `ADJUST_LOSS`
- 乐观锁和重试。
- 预留过期释放任务。
- 成品/原料库存查询。
- 库存流水查询。

关键行为：

- 可用库存不能为负。
- 销售出库扣锁定库存。
- 领料出库扣可用库存。
- 每次变更写同事务 before/after 流水。
- `total_qty = available_qty + locked_qty + qc_qty`。
- 在途库存单独跟踪。

验收：

- 预留把可用转锁定。
- 释放把锁定转可用。
- 销售出库减少锁定和实际库存。
- 质检合格把质检转可用。
- 库存不足拒绝操作。
- 并发预留不能超卖。

## 阶段 5：订单 MVP

目标：支持销售订单、生产订单和库存联动。

范围：

- 销售订单创建、编辑、列表、详情。
- `SizeMatrix` 值对象。
- 销售订单状态机。
- 销售订单审批。
- 审批通过后预留库存。
- 取消订单释放预留。
- 订单时间线。
- 库存满足率矩阵。
- 生产订单创建和生成。
- 生产订单行独立状态。
- 销售订单与生产订单多对多。
- 数量变更单可在基础生命周期之后实现，但不能遗忘。

关键行为：

- 销售订单行是颜色款级别，数量在 JSONB 尺码矩阵中。
- 已确认订单不能直接改数量。
- 生产订单必须记录具体 `bom_id`。
- 生产订单行可以跳过裁剪。
- 每次状态流转写变更日志。

验收：

- 创建矩阵订单并正确计算数量和金额。
- 提交、审批、确认流程可走通。
- 确认后预留库存或记录缺口。
- 未关联生产订单的确认订单可取消并释放库存。
- 可从缺口生成生产订单。

## 阶段 6：采购 MVP

目标：把生产需求转成物料需求和采购订单。

范围：

- BOM 管理。
- BOM 审批和版本。
- MRP 计算。
- MRP 结果和来源明细。
- 按供应商从 MRP 生成采购订单。
- 采购订单状态机。
- 采购审批。
- 采购下发创建在途库存。
- ASN 创建。
- 采购价格历史。

关键行为：

- 同一 SPU 只能有一个 APPROVED BOM。
- BOM 支持三种消耗类型。
- 面料按尺码绝对用量和损耗率计算。
- MRP 净需求 = 毛需求 - 可用库存 - 在途数量。
- 建议采购量满足 MOQ 和采购倍数。
- 重新计算生成新批次，旧的未处理结果过期。

验收：

- BOM 面料用量按尺码和损耗正确计算。
- MRP 记录来源和计算明细。
- 库存充足时不生成采购建议。
- MRP 结果按供应商生成采购订单。
- 采购下发后在途可见。

## 阶段 7：仓库基础作业

目标：先跑通收货、质检、上架、简单出库。

范围：

- ASN 收货作业。
- 分批收货。
- 批次号生成。
- 质检作业。
- 上架推荐和确认。
- 简单销售出库。
- 简单拣货。
- 复核和发货。
- 发货扣锁定库存并更新订单。
- 生产领料。
- 退料。

关键行为：

- 不同次到货生成不同批次。
- 收货使在途转质检。
- 质检合格使质检转可用。
- 质检不合格减少质检并可生成退货。
- 上架更新库存库位和库位容量。
- 发货确认才扣库存。

验收：

- 采购下发 -> ASN -> 收货 -> 质检合格 -> 原料可用。
- 销售订单预留 -> 出库 -> 拣货 -> 复核 -> 发货 -> 锁定库存扣减。
- 领料扣原料可用。
- 退料按同批次回可用库存。

## 阶段 8：仓库高级作业

目标：用波次、分拣、条码、打印提升仓库效率。

范围：

- 波次创建和释放。
- 策略：手动、时间段、订单量、物流商。
- 波次释放时分配库位和批次。
- 拣货类型：逐单、按款汇总、波次。
- 路径优化。
- 汇总拣后的分拣。
- 条码打印任务。
- 打印模板。
- HTML 模板渲染 PDF。
- 扫码枪/PDA 交互。

验收：

- 波次释放生成拣货单并排序。
- 汇总拣必须进入分拣。
- 扫码校验库位和 SKU。
- 打印接口能渲染拣货单 PDF。

## 阶段 9：盘点、预警、报表

范围：

- 盘点单。
- 明盘/盲盘。
- 库位冻结和解冻。
- 差异审核和复盘阈值。
- 差异调整走库存流水。
- 库存预警规则和定时扫描。
- 库存台账。
- 出入库流水。
- 按批次库龄分析。
- 畅滞销分析。
- 缺货统计。
- Excel 导出。

验收：

- 冻结库位禁止出入库。
- 盲盘阶段不显示系统数量。
- 差异确认写库存调整流水。
- 库龄按批次入库日期计算。
- 缺货报表能展开影响订单。

## 阶段 10：通知与外部推送

范围：

- 站内消息。
- 消息接收人。
- 外部渠道推送记录。
- 事件订阅生成通知。
- 企业微信/钉钉 Webhook。
- 通知偏好。
- 推送失败重试记录。

验收：

- 提交审批产生通知。
- 库存预警产生通知。
- 质检待办通知质检员。
- 用户偏好控制渠道。

## 阶段 11：成本与供应商对账

范围：

- 领料成本记录。
- 生产订单成本归集。
- 生产入库时计算成品加权平均成本。
- 供应商对账单。
- 对账金额按合格数量乘采购单价。
- 供应商确认/异议。
- 采购价格历史分析。

验收：

- 领料出库和 `cost_material_issue` 同事务写入。
- 生产入库按领料成本计算单位成本。
- 供应商对账排除不合格数量。

## 阶段 12：上线前加固

范围：

- 性能优化。
- 索引检查。
- `inventory_operation` 分区。
- 日终对账任务。
- Outbox 监控。
- 事件重试和异常处理。
- 备份恢复脚本。
- 安全检查。
- 扫码作业体验打磨。
- 中文业务错误提示。
- 全量回归测试。

验收：

- 50 个生产订单 MRP 不超过 10 秒。
- 库存并发扣减达标。
- 主业务流程端到端通过。
- 人工注入库存不一致时，对账能发现。

## MVP 推荐切片顺序

1. 项目骨架和共享基础设施。
2. 基础数据：尺码组、SPU/颜色/SKU、物料、供应商、客户、仓库/库位、编码规则。
3. 库存核心。
4. 销售订单创建、审批、确认、预留。
5. 生产订单生成。
6. BOM 和 MRP。
7. 采购订单下发和在途库存。
8. ASN 收货、质检合格、原料可用。
9. 领料、生产入库、销售发货。
10. 库存台账和库存流水。
