## 从销售订单生成生产订单

### 接口信息

- **接口路径**：`POST /order/sales/convert-to-production`
- **请求方法**：POST
- **权限标识**：`order:sales:convert`
- **接口说明**：从已确认的销售订单中选择若干行，按款式分组合并生成生产订单。转化后销售订单状态自动变为 PRODUCING。

### 请求参数

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| salesOrderId | Long | 是 | 销售订单ID |
| lines | List | 是 | 选中的销售订单行列表（至少1行） |
| lines[].salesOrderLineId | Long | 是 | 销售订单行ID |
| lines[].skipCutting | Boolean | 否 | 是否跳过裁剪环节（针织类默认false） |
| workshopId | Long | 否 | 车间ID |
| deadlineDate | String | 否 | 要求完工日期（格式 yyyy-MM-dd） |
| remark | String | 否 | 备注（不超过500字） |

### 请求示例

```json
{
  "salesOrderId": 1,
  "lines": [
    {"salesOrderLineId": 10, "skipCutting": false},
    {"salesOrderLineId": 11, "skipCutting": true}
  ],
  "workshopId": 5,
  "deadlineDate": "2026-07-15",
  "remark": "春季订单排产"
}
```

### 业务规则

1. **前置条件**：只有 `CONFIRMED` 状态的销售订单允许转化
2. **行归属校验**：选中的行必须属于该销售订单
3. **重复转化防护**：已全额转化的行（已分配数量 = 行总数量）不可重复转化，部分转化的行可继续转化剩余部分
4. **BOM 自动关联**：系统从 SPU 的 `defaultBomId` 获取已审批 BOM，无 BOM 时抛出异常
5. **同款合并**：同一 spuId 的多行（不同颜色）合并为一张生产订单，尺码矩阵按尺码维度累加
6. **拆分生产**：不同 spuId 的行生成不同的生产订单行，同一销售订单可通过多次转化拆分为多张生产订单
7. **状态自动流转**：转化成功后，销售订单状态通过状态机从 CONFIRMED → PRODUCING
8. **关联记录**：自动建立 `order_production_source` 多对多关联，记录每行的分配数量
9. **变更日志**：自动记录状态变更日志

### 错误码

| 错误码 | 含义 | 说明 |
|--------|------|------|
| 30007 | ORDER_NOT_CONFIRMED | 只有已确认的订单允许转化生产订单 |
| 30009 | ORDER_LINE_NOT_BELONG | 所选订单行不属于该销售订单 |
| 30006 | ORDER_ALREADY_CONVERTED | 该订单行已全部转化，不可重复转化 |
| 30008 | ORDER_SPU_NO_BOM | 该款式未配置已审批的BOM，无法生成生产订单 |
| 10009 | DATA_NOT_FOUND | 销售订单不存在 |
