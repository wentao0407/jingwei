# 销售订单状态机接口文档

> 模块：order（订单管理）
> 版本：v1.0
> 日期：2026-05-02
> 前置依赖：T-16 通用状态机引擎

---

## 一、概述

本文档描述销售订单状态机的状态流转规则、前置条件、转移动作，以及前端可用的查询接口。

销售订单状态机基于 `com.jingwei.common.statemachine.StateMachine` 引擎实现，配置为 Spring Bean（`salesOrderStateMachine`），在容器中单例存在。所有销售订单共享同一个状态机实例，状态由调用方传入。

---

## 二、状态定义

| 枚举值 | 中文标签 | 说明 |
|--------|---------|------|
| DRAFT | 草稿 | 初始状态，允许编辑订单内容 |
| PENDING_APPROVAL | 待审批 | 已提交，等待审批引擎处理 |
| REJECTED | 已驳回 | 审批不通过，可修改后重新提交 |
| CONFIRMED | 已确认 | 审批通过，已触发库存预留 |
| PRODUCING | 生产中 | 已关联生产订单，正在排产 |
| READY | 备货完成 | 所有SKU库存已满足，可发货 |
| SHIPPED | 已发货 | 已创建出库单并完成发货 |
| COMPLETED | 已完成 | 客户已签收（终态） |
| CANCELLED | 已取消 | 订单已取消（终态，不可恢复） |

---

## 三、事件定义

| 枚举值 | 中文标签 | 说明 |
|--------|---------|------|
| SUBMIT | 提交审批 | DRAFT → PENDING_APPROVAL |
| APPROVE | 审批通过 | PENDING_APPROVAL → CONFIRMED |
| REJECT | 审批驳回 | PENDING_APPROVAL → REJECTED |
| RESUBMIT | 重新提交 | REJECTED → PENDING_APPROVAL |
| START_PRODUCE | 开始排产 | CONFIRMED → PRODUCING |
| READY_STOCK | 备货完成 | PRODUCING → READY |
| SHIP | 发货 | READY → SHIPPED / PRODUCING → SHIPPED |
| SIGN_OFF | 确认签收 | SHIPPED → COMPLETED |
| CANCEL | 取消订单 | DRAFT → CANCELLED / CONFIRMED → CANCELLED |

## 五、前端查询接口

### 5.1 查询当前可执行操作

**接口路径：** `POST /order/sales/available-actions`

**说明：** 根据订单当前状态，返回该订单可执行的操作列表。前端据此动态渲染操作按钮（如"提交"、"审批通过"、"取消"等）。

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| orderId | Long | 是 | 销售订单ID |

**响应结构：**

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "event": "SUBMIT",
      "description": "提交订单审批",
      "targetStatus": "PENDING_APPROVAL"
    },
    {
      "event": "CANCEL",
      "description": "取消订单",
      "targetStatus": "CANCELLED"
    }
  ]
}
```

**各状态的可用操作：**

| 当前状态 | 可用操作 |
|---------|---------|
| DRAFT | SUBMIT（提交审批）、CANCEL（取消） |
| PENDING_APPROVAL | APPROVE（审批通过）、REJECT（审批驳回） |
| REJECTED | RESUBMIT（重新提交） |
| CONFIRMED | START_PRODUCE（开始排产）、CANCEL（取消） |
| PRODUCING | READY_STOCK（备货完成）、SHIP（部分发货） |
| READY | SHIP（发货） |
| SHIPPED | SIGN_OFF（确认签收） |
| COMPLETED | 无（终态） |
| CANCELLED | 无（终态） |

---

### 5.2 触发状态转移

**接口路径：** `POST /order/sales/fire-event`

**说明：** 对销售订单触发状态转移事件。状态机引擎会校验前置条件、执行转移动作、回调监听器。非法转移或条件不满足时抛出异常。

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| orderId | Long | 是 | 销售订单ID |
| event | String | 是 | 事件枚举值（如 SUBMIT、APPROVE、REJECT 等） |
| reason | String | 否 | 操作原因（如取消原因、驳回原因等） |

**响应结构：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "orderId": 1234567890,
    "previousStatus": "DRAFT",
    "currentStatus": "PENDING_APPROVAL",
    "event": "SUBMIT"
  }
}
```

**错误码：**

| 错误码 | 说明 |
|--------|------|
| 30001 | 当前订单状态不允许此操作 |
| 30002 | 订单至少需要一行明细 |
| 30004 | 订单已关联生产订单（取消时） |

## 六、跨模块交互

| 事件 | 交互方式 | 目标模块 | 说明 |
|------|---------|---------|------|
| ApprovalPassed / ApprovalAutoPassed | Outbox 领域事件 | 订单管理 | 由 `OrderEventListener` 消费后触发 APPROVE，并执行库存预留 |
| ApprovalRejected | Outbox 领域事件 | 订单管理 | 由 `OrderEventListener` 消费后触发 REJECT |
| APPROVE（审批通过） | 领域服务动作 | 库存管理 | 按销售行尺码矩阵精确定位 SKU 并预留库存，预留来源类型为 `SALES` |
| CANCEL（取消订单） | 领域服务动作 | 库存管理 | 释放库存预留 |
| START_PRODUCE（开始排产） | Outbox 领域事件 | 采购管理 | 通知采购模块准备物料（SalesOrderProducingEvent） |
| ProductionStocked | Outbox 领域事件 | 订单管理 | 生产订单入库后查找关联销售订单，触发 READY_STOCK |
| 所有状态变更 | 监听器回调 | 变更日志 | 记录 order_change_log |
| SUBMIT（提交审批） | 直接调用 | 审批引擎 | 调用 `/approval/submit` |

## 七、当前实现说明

| 条件/动作 | 当前实现 |
|----------|---------|
| hasOrderLines | 查询销售订单行，要求至少一行有效明细 |
| hasLinkedProductionOrder | 查询销售订单与生产订单来源关系 |
| hasNoLinkedProductionOrder | 未关联生产订单时才允许取消 |
| allStockFulfilled | 查询 `inventory_allocation` 中来源类型为 `SALES` 的预留记录，判断备货是否满足 |
| partialStockFulfilled | 至少存在一条 `SALES` 预留记录即可允许部分发货 |
| hasOutboundOrder | 出库单完成后才允许 READY 状态发货 |
| onOrderConfirmed 动作 | 按 `spuId + colorWayId + sizeId` 精确匹配 SKU，逐尺码预留库存 |
| onOrderCancelled 动作 | 释放销售订单关联的库存预留 |
| onProductionStocked 事件 | `OrderEventListener` 消费生产入库事件并尝试推进销售订单到 READY |
| 变更日志监听器 | 状态变更写入订单变更日志 |
