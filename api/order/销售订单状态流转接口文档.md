# 销售订单状态流转接口文档

> 模块：订单管理（order）
> 对应后端代码：com.jingwei.order
> 任务编号：T-20

---

## 1. 提交订单审批

将草稿订单提交至审批引擎，状态从 DRAFT 变为 PENDING_APPROVAL。若无审批配置则自动通过，状态直接变为 CONFIRMED。

- **接口路径：** `/order/sales/submit`
- **请求方法：** POST

**请求参数：**

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|---------|------|
| orderId | Long | 是 | 订单ID（URL参数 @RequestParam） |

**响应结构：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": null
}
```

**业务规则：**
- 仅 DRAFT 状态的订单允许提交
- 提交前校验订单至少有一行明细
- 提交后触发审批引擎，根据审批配置决定是否需要人工审批
- 无审批配置或配置未启用时自动通过，状态直接变为 CONFIRMED
- 审批任务分配给审批配置中指定角色下的所有用户

---

## 2. 修改后重新提交

被驳回的订单修改后重新提交审批，状态从 REJECTED 变为 PENDING_APPROVAL。

- **接口路径：** `/order/sales/resubmit`
- **请求方法：** POST

**请求参数：**

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|---------|------|
| orderId | Long | 是 | 订单ID（URL参数 @RequestParam） |

**响应结构：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": null
}
```

**业务规则：**
- 仅 REJECTED 状态的订单允许重新提交
- 重新提交前校验订单至少有一行明细
- 重新提交后触发审批引擎，流程与首次提交一致
- 被驳回的订单可修改订单内容后重新提交

---

## 3. 取消订单

取消销售订单。DRAFT 状态直接取消；CONFIRMED 状态取消需未关联生产订单。

- **接口路径：** `/order/sales/cancel`
- **请求方法：** POST

**请求参数：**

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|---------|------|
| orderId | Long | 是 | 订单ID（URL参数 @RequestParam） |

**响应结构：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": null
}
```

**业务规则：**
- DRAFT 状态可直接取消
- CONFIRMED 状态取消需未关联生产订单，取消后触发库存预留释放（Outbox 预留）
- PENDING_APPROVAL 状态不允许取消
- CANCELLED 是终态，不可恢复

---

## 附：审批通过/驳回（回调接口）

审批通过或驳回由审批引擎内部处理，审批完成后自动更新销售订单状态。

- **审批通过：** PENDING_APPROVAL → CONFIRMED，触发库存预留（Outbox 预留）
- **审批驳回：** PENDING_APPROVAL → REJECTED，业务员可修改后重新提交

审批操作接口参见 `api/approval/审批引擎接口文档.md` 中的 `/approve` 接口。

---

## 附：状态流转总览

| 当前状态 | 操作 | 目标状态 | 前置条件 | 接口 |
|---------|------|---------|---------|------|
| DRAFT | 提交审批 | PENDING_APPROVAL | 至少一行明细 | `/order/sales/submit` |
| DRAFT | 取消 | CANCELLED | 无 | `/order/sales/cancel` |
| PENDING_APPROVAL | 审批通过 | CONFIRMED | 审批引擎校验 | （审批引擎回调） |
| PENDING_APPROVAL | 审批驳回 | REJECTED | 无 | （审批引擎回调） |
| REJECTED | 重新提交 | PENDING_APPROVAL | 至少一行明细 | `/order/sales/resubmit` |
| CONFIRMED | 取消 | CANCELLED | 未关联生产订单 | `/order/sales/cancel` |

**每次状态变更自动记录在 `t_order_change_log` 中。**
