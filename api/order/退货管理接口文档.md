# 退货管理接口文档

> 模块：order（退货管理）
> 版本：v1.0
> 日期：2026-05-05
> 前置依赖：order 模块（销售订单）、approval 模块（审批引擎）、inventory 模块（库存变更）

---

## 一、概述

退货管理支持三种退货场景：
| 退货类型 | 编码 | 触发场景 |
|---------|------|---------|
| 客户退货 | CUSTOMER_REJECT | 客户收货后因质量/尺码问题退回 |
| 物流拒收 | LOGISTICS_REJECT | 发货后客户拒收或无法送达 |
| 经销商退货 | DISTRIBUTOR_RETURN | 经销商退回未售出的库存（换季退货） |

退货流程：创建退货单 → 提交审批 → 审批通过 → 仓库收货 → 质检 → 完成

## 二、退货单管理

### 2.1 创建退货单

**接口路径：** `POST /order/return/create`
**权限标识：** `order:return:create`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| returnType | String | 是 | 退货类型：CUSTOMER_REJECT/LOGISTICS_REJECT/DISTRIBUTOR_RETURN |
| salesOrderId | Long | 是 | 原销售订单ID |
| salesOrderNo | String | 是 | 原销售订单编号 |
| customerId | Long | 是 | 客户ID |
| reason | String | 否 | 退货原因 |
| remark | String | 否 | 备注 |
| lines | List | 是 | 退货行列表 |

**退货行（lines）字段：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| salesOrderLineId | Long | 是 | 原销售订单行ID |
| spuId | Long | 是 | 款式ID |
| colorWayId | Long | 是 | 颜色款ID |
| sizeMatrixJson | String | 是 | 退货尺码矩阵（JSON字符串） |
| totalQuantity | Integer | 是 | 本行退货数量 |
| remark | String | 否 | 备注 |

**响应结构：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "returnNo": "RT-20260505-0001",
    "returnType": "CUSTOMER_REJECT",
    "returnTypeLabel": "客户退货",
    "salesOrderId": 100,
    "salesOrderNo": "SO-20260501",
    "customerId": 1,
    "reason": "尺码不合适",
    "status": "DRAFT",
    "statusLabel": "草稿",
    "totalQuantity": 10,
    "lines": [...]
  }
}
```

**业务规则：**
- 退货单号由编码规则引擎自动生成（格式：RT-年月日-4位流水号）
- 初始状态为 DRAFT
- 退货行不能为空
- 退货数量不能超过原订单对应 SKU 的已发货数量（累计校验）

### 2.2 查询退货单详情

**接口路径：** `POST /order/return/detail`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| returnId | Long | 是 | 退货单ID |

**响应结构：** 同创建接口响应，包含退货行列表。

### 2.3 提交退货审批

**接口路径：** `POST /order/return/submit`
**权限标识：** `order:return:submit`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| returnId | Long | 是 | 退货单ID |

**业务规则：**
- 只有 DRAFT 状态允许提交
- 提交后状态变为 PENDING_APPROVAL
- 如果无审批配置或审批人就是提交人，自动通过变为 APPROVED

### 2.4 审批通过

**接口路径：** `POST /order/return/approve`
**权限标识：** `order:return:approve`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| returnId | Long | 是 | 退货单ID |

**业务规则：**
- 只有 PENDING_APPROVAL 状态允许审批
- 审批通过后状态变为 APPROVED，记录审批人和审批时间

### 2.5 审批驳回

**接口路径：** `POST /order/return/reject`
**权限标识：** `order:return:approve`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| returnId | Long | 是 | 退货单ID |

**业务规则：**
- 只有 PENDING_APPROVAL 状态允许驳回
- 驳回后状态变为 REJECTED（终态）

### 2.6 退货收货确认

**接口路径：** `POST /order/return/receive`
**权限标识：** `order:return:receive`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| returnId | Long | 是 | 退货单ID |

**业务规则：**
- 只有 APPROVED 状态允许收货
- 收货后触发库存 INBOUND_RETURN 操作（质检库存增加）
- 收货完成后状态变为 QC

### 2.7 退货质检

**接口路径：** `POST /order/return/qc`
**权限标识：** `order:return:qc`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| returnId | Long | 是 | 退货单ID |
| results | List | 是 | 质检结果列表 |

**质检结果（results）字段：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| lineId | Long | 是 | 退货行ID |
| passedQty | Integer | 是 | 合格数量 |
| failedQty | Integer | 是 | 不合格数量 |

**业务规则：**
- 只有 QC 状态允许质检
- 合格品触发 QC_PASS 操作（质检库存→可用库存）
- 不合格品触发 QC_FAIL 操作（质检库存减少，不进入可用库存）
- 质检完成后状态变为 COMPLETED

## 三、状态流转图

```
DRAFT → PENDING_APPROVAL → APPROVED → QC → COMPLETED
                ↓
            REJECTED
```

## 四、错误码

| 错误码 | 说明 |
|--------|------|
| 30001 | 订单状态流转不合法 |
| 30002 | 订单行不能为空 |
| 30010 | 退货数量超过原订单已发货数量 |
| 30011 | 退货单状态不允许此操作 |
