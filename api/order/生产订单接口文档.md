## 1. 创建生产订单

**接口路径：** `POST /order/production/create`
**权限标识：** `order:production:create`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sourceType | String | 是 | 来源类型：MANUAL=独立创建，SALES_ORDER=从销售订单转化 |
| planDate | String | 否 | 计划生产日期（格式：yyyy-MM-dd） |
| deadlineDate | String | 否 | 要求完工日期（格式：yyyy-MM-dd） |
| workshopId | Long | 否 | 车间ID |
| remark | String | 否 | 备注（最长500字） |
| lines | Array | 是 | 订单行列表（至少1行） |
| lines[].spuId | Long | 是 | 款式ID |
| lines[].colorWayId | Long | 是 | 颜色款ID |
| lines[].sizeGroupId | Long | 是 | 尺码组ID |
| lines[].sizes | Array | 是 | 尺码数量列表 |
| lines[].sizes[].sizeId | Long | 是 | 尺码ID |
| lines[].sizes[].code | String | 是 | 尺码编码（如 S/M/L） |
| lines[].sizes[].quantity | int | 否 | 数量（默认0） |
| lines[].bomId | Long | 否 | BOM ID（下达时必须有，创建时可选） |
| lines[].skipCutting | Boolean | 否 | 是否跳过裁剪（针织类默认false） |
| lines[].remark | String | 否 | 行备注 |

### 请求示例

```json
{
  "sourceType": "MANUAL",
  "planDate": "2026-05-10",
  "deadlineDate": "2026-06-01",
  "workshopId": 5,
  "remark": "春季大货生产",
  "lines": [
    {
      "spuId": 201,
      "colorWayId": 301,
      "sizeGroupId": 1,
      "sizes": [
        {"sizeId": 10, "code": "S", "quantity": 100},
        {"sizeId": 11, "code": "M", "quantity": 200},
        {"sizeId": 12, "code": "L", "quantity": 300}
      ],
      "bomId": 501,
      "skipCutting": false
    },
    {
      "spuId": 202,
      "colorWayId": 302,
      "sizeGroupId": 1,
      "sizes": [
        {"sizeId": 10, "code": "S", "quantity": 50},
        {"sizeId": 11, "code": "M", "quantity": 100},
        {"sizeId": 12, "code": "L", "quantity": 150}
      ],
      "skipCutting": true
    }
  ]
}
```

### 响应结构

返回 `R<ProductionOrderVO>`，ProductionOrderVO 结构见第6节。

### 业务规则

- 订单编号由编码规则自动生成（格式：MO-年月-5位流水号），不可手动指定
- 同一订单内不允许重复的款式+颜色组合（spuId + colorWayId 唯一）
- 行总数量由尺码矩阵自动汇总，无需手动传入
- 新建订单状态默认 DRAFT
- 每行有独立状态，默认 DRAFT
- 主表状态取所有行的最滞后状态（按状态枚举 ordinal 最小值）
- skip_cutting 标记：针织类面料跳过裁剪工序

## 2. 编辑草稿订单

**接口路径：** `POST /order/production/update`
**权限标识：** `order:production:update`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderId | Long（Query） | 是 | 订单ID |
| planDate | String | 否 | 计划生产日期 |
| deadlineDate | String | 否 | 要求完工日期 |
| workshopId | Long | 否 | 车间ID |
| remark | String | 否 | 备注 |
| lines | Array | 是 | 订单行列表（全量替换） |

lines 结构同创建接口。

### 业务规则

- 仅 DRAFT 状态的订单允许编辑
- 采用全量替换策略：传入完整行列表，后端先删旧行再插新行
- 非DRAFT状态编辑返回错误码 10010（当前状态不允许此操作）
- 编辑时同样校验行唯一性

## 3. 删除草稿订单

**接口路径：** `POST /order/production/delete`
**权限标识：** `order:production:delete`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderId | Long（Query） | 是 | 订单ID |

### 业务规则

- 仅 DRAFT 状态的订单允许删除
- 删除时同时删除所有订单行

## 4. 查询订单详情

**接口路径：** `POST /order/production/detail`
**权限标识：** 无（登录即可访问）

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderId | Long（Query） | 是 | 订单ID |

### 响应说明

返回完整的订单信息，包括：
- 订单主表字段
- 订单行列表（含尺码矩阵）
- 补充款式名称/编码、颜色名称/编码等冗余展示信息

## 5. 分页查询生产订单

**接口路径：** `POST /order/production/page`
**权限标识：** 无（登录即可访问）

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | Integer | 否 | 当前页码（默认1） |
| size | Integer | 否 | 每页条数（默认10） |
| status | String | 否 | 状态筛选（DRAFT/RELEASED/PLANNED/CUTTING/SEWING/FINISHING/COMPLETED/STOCKED） |
| orderNo | String | 否 | 订单编号搜索（模糊匹配） |
| planDateStart | String | 否 | 计划日期起始（格式：yyyy-MM-dd） |
| planDateEnd | String | 否 | 计划日期结束（格式：yyyy-MM-dd） |

## 7. 生产订单状态枚举

| 状态编码 | 中文标签 | 顺序 | 说明 |
|----------|----------|------|------|
| DRAFT | 草稿 | 0 | 新建默认状态 |
| RELEASED | 已下达 | 1 | 下达到车间 |
| PLANNED | 已排产 | 2 | 排入生产计划 |
| CUTTING | 裁剪中 | 3 | 裁剪工序进行中 |
| SEWING | 缝制中 | 4 | 缝制工序进行中 |
| FINISHING | 后整中 | 5 | 后整工序进行中 |
| COMPLETED | 已完工 | 6 | 生产完成 |
| STOCKED | 已入库 | 7 | 成品入库完成 |

---

## 8. 错误码

| 错误码 | 说明 |
|--------|------|
| 30002 | 订单至少需要一行明细（ORDER_LINE_EMPTY） |
| 30003 | 同一订单中不能添加重复的款式和颜色（ORDER_LINE_DUPLICATE） |
| 10010 | 当前状态不允许此操作（非DRAFT编辑/删除） |
| 10009 | 数据不存在（订单不存在） |
| 10002 | 参数校验失败 |
