# 生产订单接口文档

> 模块：订单管理（order）
> 对应后端代码：com.jingwei.order

## 1. 创建生产订单

**接口路径：** `POST /order/production/create`
**权限标识：** `order:production:create`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sourceType | String | 是 | 来源类型：MANUAL=独立创建，SALES_ORDER=从销售订单转化 |
| planDate | String | 否 | 计划生产日期（格式：yyyy-MM-dd） |
| deadlineDate | String | 否 | 要求完工日期（格式：yyyy-MM-dd） |
| workshopId | Long | 否 | 车间ID |
| remark | String | 否 | 备注（最长500字） |
| lines | Array | 是 | 订单行列表（至少1行） |
| lines[].spuId | Long | 是 | 款式ID |
| lines[].colorWayId | Long | 是 | 颜色款ID |
| lines[].sizeGroupId | Long | 是 | 尺码组ID |
| lines[].sizes | Array | 是 | 尺码数量列表 |
| lines[].sizes[].sizeId | Long | 是 | 尺码ID |
| lines[].sizes[].code | String | 是 | 尺码编码（如 S/M/L） |
| lines[].sizes[].quantity | int | 否 | 数量（默认0） |
| lines[].bomId | Long | 否 | BOM ID（下达时必须有，创建时可选） |
| lines[].skipCutting | Boolean | 否 | 是否跳过裁剪（针织类默认false） |
| lines[].remark | String | 否 | 行备注 |

### 请求示例

```json
{
  "sourceType": "MANUAL",
  "planDate": "2026-05-10",
  "deadlineDate": "2026-06-01",
  "workshopId": 5,
  "remark": "春季大货生产",
  "lines": [
    {
      "spuId": 201,
      "colorWayId": 301,
      "sizeGroupId": 1,
      "sizes": [
        {"sizeId": 10, "code": "S", "quantity": 100},
        {"sizeId": 11, "code": "M", "quantity": 200},
        {"sizeId": 12, "code": "L", "quantity": 300}
      ],
      "bomId": 501,
      "skipCutting": false
    },
    {
      "spuId": 202,
      "colorWayId": 302,
      "sizeGroupId": 1,
      "sizes": [
        {"sizeId": 10, "code": "S", "quantity": 50},
        {"sizeId": 11, "code": "M", "quantity": 100},
        {"sizeId": 12, "code": "L", "quantity": 150}
      ],
      "skipCutting": true
    }
  ]
}
```

### 响应结构

返回 `R<ProductionOrderVO>`，ProductionOrderVO 结构见第6节。

### 业务规则

- 订单编号由编码规则自动生成（格式：MO-年月-5位流水号），不可手动指定
- 同一订单内不允许重复的款式+颜色组合（spuId + colorWayId 唯一）
- 行总数量由尺码矩阵自动汇总，无需手动传入
- 新建订单状态默认 DRAFT
- 每行有独立状态，默认 DRAFT
- 主表状态取所有行的最滞后状态（按状态枚举 ordinal 最小值）
- skip_cutting 标记：针织类面料跳过裁剪工序

## 2. 编辑草稿订单

**接口路径：** `POST /order/production/update`
**权限标识：** `order:production:update`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderId | Long（Query） | 是 | 订单ID |
| planDate | String | 否 | 计划生产日期 |
| deadlineDate | String | 否 | 要求完工日期 |
| workshopId | Long | 否 | 车间ID |
| remark | String | 否 | 备注 |
| lines | Array | 是 | 订单行列表（全量替换） |

lines 结构同创建接口。

### 业务规则

- 仅 DRAFT 状态的订单允许编辑
- 采用全量替换策略：传入完整行列表，后端先删旧行再插新行
- 非DRAFT状态编辑返回错误码 10010（当前状态不允许此操作）
- 编辑时同样校验行唯一性

## 7. 生产订单状态枚举

| 状态编码 | 中文标签 | 顺序 | 说明 |
|----------|----------|------|------|
| DRAFT | 草稿 | 0 | 新建默认状态 |
| RELEASED | 已下达 | 1 | 下达到车间 |
| PLANNED | 已排产 | 2 | 排入生产计划 |
| CUTTING | 裁剪中 | 3 | 裁剪工序进行中 |
| SEWING | 缝制中 | 4 | 缝制工序进行中 |
| FINISHING | 后整中 | 5 | 后整工序进行中 |
| COMPLETED | 已完工 | 6 | 生产完成 |
| STOCKED | 已入库 | 7 | 成品入库完成 |

# 生产订单状态机流转接口（T-23）

> 任务编号：T-23
> 实现日期：2026-05-04
> 依赖：通用状态机引擎（T-16）

## 一、概述

生产订单状态机基于 `com.jingwei.common.statemachine.StateMachine` 引擎实现，支持主表级别和行级别的状态流转。

**状态流转图：**
```
DRAFT → RELEASED → PLANNED → CUTTING → SEWING → FINISHING → COMPLETED → STOCKED
                    └──────────────────→ SEWING（跳过裁剪）
```

**核心特性：**
- 每行有独立状态，主表状态取所有行的最滞后状态
- 针织类可跳过裁剪（skipCutting=true）
- 状态变更自动记录到 t_order_change_log
- FINISHING → COMPLETED 时通知库存模块准备入库
- COMPLETED → STOCKED 时通知销售订单可发货

---

## 二、触发主表状态转移

**接口路径：** `POST /order/production/fire-event`
**权限标识：** `order:production:fire-event`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderId | Long | 是 | 订单ID |
| event | String | 是 | 事件编码（见事件枚举） |

### 事件枚举

| 事件编码 | 中文标签 | 适用状态 | 说明 |
|----------|----------|----------|------|
| RELEASE | 下达生产订单 | DRAFT | 前提：所有行有BOM和数量 |
| PLAN | 排产完成 | RELEASED | - |
| START_CUTTING | 开始裁剪 | PLANNED | 前提：不跳过裁剪 |
| START_SEWING | 开始缝制 | PLANNED/CUTTING | PLANNED时需skipCutting=true |
| START_FINISHING | 开始后整 | SEWING | - |
| COMPLETE | 生产完工 | FINISHING | 触发通知库存准备入库 |
| STOCK_IN | 入库完成 | COMPLETED | 前提：全部入库，通知销售可发货 |

### 请求示例

```json
{
  "orderId": 1,
  "event": "RELEASE"
}
```

### 响应结构

返回 `R<Void>`，成功时无返回数据。

### 业务规则

- 状态转移不合法时抛出 StateMachineException
- 前置条件不满足时抛出 BizException（如缺少BOM）
- 每次转移自动记录变更日志到 t_order_change_log

---

## 三、触发行级别状态转移

**接口路径：** `POST /order/production/fire-line-event`
**权限标识：** `order:production:fire-line-event`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderId | Long | 是 | 订单ID |
| lineId | Long | 是 | 行ID |
| event | String | 是 | 事件编码（同主表事件枚举） |

### 请求示例

```json
{
  "orderId": 1,
  "lineId": 10,
  "event": "START_CUTTING"
}
```

### 响应结构

返回 `R<Void>`，成功时无返回数据。

### 业务规则

- 行状态变更后自动重新计算主表状态（取所有行最滞后状态）
- 行级别支持独立的状态流转，允许同一订单不同行处于不同阶段
- skipCutting 逻辑：START_CUTTING 需 skipCutting=false，START_SEWING 在 PLANNED 状态需 skipCutting=true

## 六、状态转移规则详细说明

### 6.1 DRAFT → RELEASED（下达）

**前置条件：**
- 所有行必须有 bomId（不为空）
- 所有行必须有 totalQuantity（> 0）

**失败示例：**
```json
{
  "code": 30005,
  "message": "第1行未关联BOM，无法下达"
}
```

### 6.2 PLANNED → CUTTING / SEWING（跳过裁剪）

**逻辑说明：**
- START_CUTTING：仅当 skipCutting=false 时允许
- START_SEWING：在 PLANNED 状态时，仅当 skipCutting=true 时允许（跳过裁剪直接缝制）
- START_SEWING：在 CUTTING 状态时，无条件允许（裁剪完成进入缝制）

**典型场景：**
- 梭织面料：PLANNED → CUTTING → SEWING
- 针织面料：PLANNED → SEWING（跳过裁剪）

### 6.3 FINISHING → COMPLETED（完工）

**触发动作：**
- 发布 ProductionCompletedEvent 通知库存模块准备入库
- 当前阶段（T-40前）仅打日志

### 6.4 COMPLETED → STOCKED（入库）

**前置条件：**
- 所有行的已入库数量 = 总数量（当前阶段预留，待库存模块实现）

**触发动作：**
- 发布 ProductionStockedEvent 通知销售订单可发货
- 当前阶段（T-40前）仅打日志
