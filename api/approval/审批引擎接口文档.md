# 审批引擎接口文档

> 模块：approval（审批引擎）
> 版本：v1.0
> 日期：2026-05-02
> 前置依赖：T-16 通用状态机引擎、system 模块（用户/角色）

---

## 一、概述

审批引擎解决"谁有权触发状态流转"的问题，与状态机配合使用。支持两种模式：
- **单人审批（SINGLE）**：指定一个角色，该角色下任一用户审批即可
- **或签（OR_SIGN）**：指定多个角色，任意一人审批即通过，其他人待办自动取消

业务模块通过调用 `POST /approval/submit` 提交审批，审批结果通过领域事件（Outbox）通知业务模块。

## 二、审批配置管理

### 2.1 创建审批配置

**接口路径：** `POST /approval/config/create`
**权限标识：** `system:approvalConfig:create`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| businessType | String | 是 | 业务类型（如 SALES_ORDER、PURCHASE_ORDER） |
| configName | String | 是 | 配置名称 |
| approvalMode | String | 是 | 审批模式：SINGLE / OR_SIGN |
| approverRoleIds | List\<Long\> | 是 | 审批人角色ID列表，单人模式只能含1个 |
| enabled | Boolean | 否 | 是否启用，默认 true |

**响应结构：**
```json
{ "code": 0, "message": "success", "data": { "id": 1, "businessType": "SALES_ORDER", ... } }
```

**业务规则：**
- 同一业务类型只能有一条审批配置
- 单人审批模式只能指定一个角色
- 或签模式可指定多个角色

### 2.2 更新审批配置

**接口路径：** `POST /approval/config/update`
**权限标识：** `system:approvalConfig:update`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 配置ID |
| configName | String | 否 | 配置名称 |
| approvalMode | String | 否 | 审批模式 |
| approverRoleIds | List\<Long\> | 否 | 审批人角色ID列表 |
| enabled | Boolean | 否 | 是否启用 |

### 2.3 删除审批配置

**接口路径：** `POST /approval/config/delete`
**权限标识：** `system:approvalConfig:delete`

**请求参数：** Long configId

### 2.4 查询审批配置详情

**接口路径：** `POST /approval/config/detail`

**请求参数：** Long configId

### 2.5 查询所有审批配置列表

**接口路径：** `POST /approval/config/list`

## 三、审批操作

### 3.1 提交审批

**接口路径：** `POST /approval/submit`

**说明：** 由业务模块（如销售订单）调用，将业务单据提交到审批引擎。返回 true 表示需要人工审批，false 表示自动通过（无配置或配置未启用时）。

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| businessType | String | 是 | 业务类型 |
| businessId | Long | 是 | 业务单据ID |
| businessNo | String | 是 | 业务单据编号 |

**响应结构：**
```json
{ "code": 0, "message": "success", "data": true }
```

**业务规则：**
- 无审批配置 → 自动通过，返回 false
- 配置未启用 → 自动通过，返回 false
- 审批角色下无用户 → 自动通过，返回 false
- 单人审批 → 为该角色下所有用户创建待办，任一人审批后其他待办自动取消
- 或签 → 为所有角色下所有用户创建待办

### 3.2 审批操作

**接口路径：** `POST /approval/approve`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| taskId | Long | 是 | 审批任务ID |
| approved | Boolean | 是 | 是否通过 |
| opinion | String | 是 | 审批意见（必填） |

**业务规则：**
- 只有持有待办的审批人能审批，其他人 → 错误码 70002
- 审批意见为空 → 错误码 70003
- 任务已处理 → 错误码 10010
- 或签模式下一人审批后，其他待办自动取消
- 审批结果通过 Outbox 发布领域事件（ApprovalPassed / ApprovalRejected）

## 四、审批查询

### 4.1 查询当前用户待办

**接口路径：** `POST /approval/task/myPending`

**响应结构：**
```json
{
  "code": 0, "data": [
    { "id": 1, "businessType": "SALES_ORDER", "businessId": 123, "businessNo": "SO-001",
      "approvalMode": "SINGLE", "status": "PENDING", "approverId": 10, "createdAt": "..." }
  ]
}
```

### 4.2 查询业务单据审批记录

**接口路径：** `POST /approval/task/records`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| businessType | String | 是 | 业务类型 |
| businessId | Long | 是 | 业务单据ID |

## 五、领域事件

| 事件类型 | 触发时机 | 事件数据 |
|---------|---------|---------|
| ApprovalPassed | 审批通过 | businessType, businessId, businessNo, taskId |
| ApprovalRejected | 审批驳回 | businessType, businessId, businessNo, taskId |

业务模块订阅示例（T-20 实现后）：
```java
@EventListener
public void onApprovalPassed(ApprovalPassedEvent event) {
    if ("SALES_ORDER".equals(event.getBusinessType())) {
        salesOrderDomainService.onApproved(event.getBusinessId());
    }
}
```
