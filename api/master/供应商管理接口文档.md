# 供应商管理接口文档

模块路径：`com.jingwei.master`  
Controller：`SupplierController`

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串，如 `?supplierId=1`）；JSON Body = `@RequestBody`（请求体 JSON）  
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

---

## 1. 创建供应商

**接口路径**：`POST /master/supplier/create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 是 | 供应商名称，最长128字符，全局唯一 |
| shortName | String | 否 | 简称，最长64字符 |
| type | String | 是 | 供应商类型：FABRIC/TRIM/PACKAGING/COMPOSITE |
| contactPerson | String | 否 | 联系人，最长32字符 |
| contactPhone | String | 否 | 联系电话，最长20字符 |
| address | String | 否 | 地址 |
| settlementType | String | 否 | 结算方式：MONTHLY/QUARTERLY/COD |
| leadTimeDays | Integer | 否 | 平均交货天数 |
| remark | String | 否 | 备注 |

**响应结构**：`R<SupplierVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 供应商ID |
| data.code | String | 供应商编码（自动生成，格式 SUP-000001） |
| data.name | String | 供应商名称 |
| data.shortName | String | 简称 |
| data.type | String | 供应商类型：FABRIC/TRIM/PACKAGING/COMPOSITE |
| data.contactPerson | String | 联系人 |
| data.contactPhone | String | 联系电话 |
| data.address | String | 地址 |
| data.settlementType | String | 结算方式：MONTHLY/QUARTERLY/COD |
| data.leadTimeDays | Integer | 平均交货天数 |
| data.qualificationStatus | String | 资质状态：QUALIFIED/PENDING/DISQUALIFIED（新建默认 PENDING） |
| data.status | String | 状态：ACTIVE/INACTIVE（新建默认 ACTIVE） |
| data.remark | String | 备注 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 供应商编码由编码规则引擎自动生成（格式：SUP-6位流水号），不可手动指定
- 供应商名称全局唯一
- 创建后默认状态为 ACTIVE，资质状态为 PENDING

---

## 2. 更新供应商

**接口路径**：`POST /master/supplier/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supplierId | Long | 是 | Query参数，供应商ID |
| name | String | 否 | 供应商名称，最长128字符（修改时校验唯一性） |
| shortName | String | 否 | 简称，最长64字符 |
| contactPerson | String | 否 | 联系人，最长32字符 |
| contactPhone | String | 否 | 联系电话，最长20字符 |
| address | String | 否 | 地址 |
| settlementType | String | 否 | 结算方式：MONTHLY/QUARTERLY/COD |
| leadTimeDays | Integer | 否 | 平均交货天数 |
| qualificationStatus | String | 否 | 资质状态：QUALIFIED/PENDING/DISQUALIFIED |
| remark | String | 否 | 备注 |

**响应结构**：`R<SupplierVO>`

**业务规则**：
- 编码（code）和类型（type）不可修改——编码被业务单据引用，类型影响采购策略和 BOM 关联
- 名称修改时校验全局唯一性（排除自身）
- 所有字段可选，只更新传入的字段

---

## 3. 停用供应商

**接口路径**：`POST /master/supplier/deactivate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supplierId | Long | 是 | Query参数，供应商ID |

**响应结构**：`R<Void>`

**业务规则**：
- 停用后不可创建新采购订单，已有采购订单不受影响
- 已停用的供应商重复停用返回错误

---

## 4. 启用供应商

**接口路径**：`POST /master/supplier/activate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supplierId | Long | 是 | Query参数，供应商ID |

**响应结构**：`R<Void>`

**业务规则**：
- 将停用的供应商重新启用，启用后可创建新采购订单
- 已启用的供应商重复启用返回错误

---

## 5. 删除供应商

**接口路径**：`POST /master/supplier/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supplierId | Long | 是 | Query参数，供应商ID |

**响应结构**：`R<Void>`

**业务规则**：
- 仅允许删除未被采购订单引用的供应商
- 已被引用时返回错误提示（当前采购模块未实现，预留钩子）
- 逻辑删除，不物理删除

---

## 7. 分页查询供应商

**接口路径**：`POST /master/supplier/page`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | Long | 否 | 当前页码，默认1 |
| size | Long | 否 | 每页条数，默认10 |
| type | String | 否 | 供应商类型筛选：FABRIC/TRIM/PACKAGING/COMPOSITE |
| qualificationStatus | String | 否 | 资质状态筛选：QUALIFIED/PENDING/DISQUALIFIED |
| status | String | 否 | 状态筛选：ACTIVE/INACTIVE |
| keyword | String | 否 | 关键词搜索（匹配编码或名称） |

**响应结构**：`R<IPage<SupplierVO>>`

分页结构包含 records（数据列表）、total（总条数）、current（当前页码）、size（每页条数）等字段。

---

## 8. 供应商相关错误码

| 错误码 | 说明 |
|--------|------|
| 20010 | 供应商名称已存在（SUPPLIER_NAME_DUPLICATE） |
| 20011 | 供应商资质不合格，不可创建采购订单（SUPPLIER_DISQUALIFIED） |
| 10008 | 数据已存在（编码重复等通用场景） |
| 10009 | 数据不存在 |
| 10010 | 当前状态不允许此操作（如已停用再停用） |
| 10011 | 并发冲突，数据已被他人修改 |
