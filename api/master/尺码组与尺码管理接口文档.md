# 尺码组与尺码管理接口文档

模块路径：`com.jingwei.master`（尺码组部分）  
Controller：`SizeGroupController`

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串，如 `?sizeGroupId=1`）；JSON Body = `@RequestBody`（请求体 JSON）  
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

---

## 1. 尺码组接口

### 1.1 创建尺码组

**接口路径**：`POST /master/size-group/create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | String | 是 | 尺码组编码，最长32字符，全局唯一 |
| name | String | 是 | 尺码组名称，最长64字符 |
| category | String | 是 | 适用品类：WOMEN/MEN/CHILDREN |

**响应结构**：`R<SizeGroupVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 尺码组ID |
| data.code | String | 尺码组编码 |
| data.name | String | 尺码组名称 |
| data.category | String | 适用品类：WOMEN/MEN/CHILDREN |
| data.status | String | 状态：ACTIVE/INACTIVE |
| data.sizes | List\<SizeVO\> | 尺码列表（创建时为空） |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 编码全局唯一
- 创建后默认状态为 ACTIVE
- 创建时尺码列表为空，需单独添加尺码

### 1.2 更新尺码组

**接口路径**：`POST /master/size-group/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sizeGroupId | Long | 是 | Query参数，尺码组ID |
| name | String | 否 | 尺码组名称，最长64字符 |
| category | String | 否 | 适用品类：WOMEN/MEN/CHILDREN |
| status | String | 否 | 状态：ACTIVE/INACTIVE |

**响应结构**：`R<SizeGroupVO>`

**业务规则**：
- 尺码组编码不可修改
- 所有字段可选，只更新传入的字段

### 1.3 删除尺码组

**接口路径**：`POST /master/size-group/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sizeGroupId | Long | 是 | Query参数，尺码组ID |

**响应结构**：`R<Void>`

**业务规则**：
- 被SPU引用的尺码组不可删除，但可停用
- 删除时同时删除组内所有尺码

### 1.4 查询尺码组列表

**接口路径**：`POST /master/size-group/list`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| category | String | 否 | 适用品类筛选：WOMEN/MEN/CHILDREN |
| status | String | 否 | 状态筛选 |

**响应结构**：`R<List<SizeGroupVO>>`

列表结果不含尺码详情（sizes 为 null 或空），需通过详情接口获取。

### 1.5 查询尺码组详情

**接口路径**：`POST /master/size-group/detail`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sizeGroupId | Long | 是 | Query参数，尺码组ID |

**响应结构**：`R<SizeGroupVO>`

详情包含尺码列表（sizes），按 sort_order 排序。

## 2. 尺码接口

### 2.1 新增尺码

**接口路径**：`POST /master/size-group/size/create`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sizeGroupId | Long | 是 | Query参数，尺码组ID |
| code | String | 是 | 尺码编码，最长16字符，组内唯一 |
| name | String | 是 | 尺码名称，最长16字符 |
| sortOrder | Integer | 否 | 排序号，不传则追加到末尾 |

**响应结构**：`R<SizeVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 尺码ID |
| data.sizeGroupId | Long | 所属尺码组ID |
| data.code | String | 尺码编码 |
| data.name | String | 尺码名称 |
| data.sortOrder | Integer | 排序号 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 同一尺码组内编码不可重复
- 已被引用的尺码组也允许新增尺码（追加到末尾）

### 2.2 更新尺码

**接口路径**：`POST /master/size-group/size/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sizeId | Long | 是 | Query参数，尺码ID |
| code | String | 否 | 尺码编码，最长16字符 |
| name | String | 否 | 尺码名称，最长16字符 |
| sortOrder | Integer | 否 | 排序号 |

**响应结构**：`R<SizeVO>`

**业务规则**：
- 已被SPU引用的尺码组内，不允许修改尺码编码
- 所有字段可选，只更新传入的字段

### 2.3 删除尺码

**接口路径**：`POST /master/size-group/size/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sizeId | Long | 是 | Query参数，尺码ID |

**响应结构**：`R<Void>`

**业务规则**：
- 已被SPU引用的尺码组内，不允许删除尺码
