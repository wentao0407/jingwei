# 仓库与库位管理接口文档

模块路径：`com.jingwei.master`  
Controller：`WarehouseController`

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串）；JSON Body = `@RequestBody`（请求体 JSON）  
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

---

## 1. 仓库接口

### 1.1 创建仓库

**接口路径**：`POST /master/warehouse/create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | String | 是 | 仓库编码，最长16字符，全局唯一（如 WH01） |
| name | String | 是 | 仓库名称，最长64字符 |
| type | String | 是 | 仓库类型：FINISHED_GOODS/RAW_MATERIAL/RETURN |
| address | String | 否 | 地址 |
| managerId | Long | 否 | 仓库管理员ID |
| remark | String | 否 | 备注 |

**响应结构**：`R<WarehouseVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 仓库ID |
| data.code | String | 仓库编码 |
| data.name | String | 仓库名称 |
| data.type | String | 仓库类型：FINISHED_GOODS/RAW_MATERIAL/RETURN |
| data.address | String | 地址 |
| data.managerId | Long | 仓库管理员ID |
| data.status | String | 状态：ACTIVE/INACTIVE（新建默认 ACTIVE） |
| data.remark | String | 备注 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 仓库编码手动指定，全局唯一
- 创建后默认状态为 ACTIVE

### 1.2 更新仓库

**接口路径**：`POST /master/warehouse/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| warehouseId | Long | 是 | Query参数，仓库ID |
| name | String | 否 | 仓库名称 |
| address | String | 否 | 地址 |
| managerId | Long | 否 | 仓库管理员ID |
| remark | String | 否 | 备注 |

**响应结构**：`R<WarehouseVO>`

**业务规则**：
- 编码（code）和类型（type）不可修改

### 1.3 停用仓库

**接口路径**：`POST /master/warehouse/deactivate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| warehouseId | Long | 是 | Query参数，仓库ID |

**响应结构**：`R<Void>`

**业务规则**：
- 停用后其下库位不可选用，已有库存不受影响
- 已停用的仓库重复停用返回错误

### 1.4 启用仓库

**接口路径**：`POST /master/warehouse/activate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| warehouseId | Long | 是 | Query参数，仓库ID |

**响应结构**：`R<Void>`

### 1.5 删除仓库

**接口路径**：`POST /master/warehouse/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| warehouseId | Long | 是 | Query参数，仓库ID |

**响应结构**：`R<Void>`

**业务规则**：
- 同时删除仓库下的所有库位
- 存在库存数据时不可删除（预留钩子）

### 1.6 查询仓库详情（含库位列表）

**接口路径**：`POST /master/warehouse/detail`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| warehouseId | Long | 是 | Query参数，仓库ID |

**响应结构**：`R<WarehouseVO>`（含 locations 列表）

### 1.7 查询仓库列表

**接口路径**：`POST /master/warehouse/list`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | String | 否 | 仓库类型筛选 |
| status | String | 否 | 状态筛选：ACTIVE/INACTIVE |

**响应结构**：`R<List<WarehouseVO>>`（不含库位列表）

### 1.8 分页查询仓库

**接口路径**：`POST /master/warehouse/page`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | Long | 否 | 当前页码，默认1 |
| size | Long | 否 | 每页条数，默认10 |
| type | String | 否 | 仓库类型筛选 |
| status | String | 否 | 状态筛选 |
| keyword | String | 否 | 关键词搜索（匹配编码或名称） |

**响应结构**：`R<IPage<WarehouseVO>>`

## 2. 库位接口

### 2.1 新增库位

**接口路径**：`POST /master/warehouse/location/create`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| warehouseId | Long | 是 | Query参数，仓库ID |
| zoneCode | String | 是 | 库区编码（如 A），最长16字符 |
| rackCode | String | 是 | 货架编码（如 01），最长16字符 |
| rowCode | String | 是 | 层编码（如 02），最长16字符 |
| binCode | String | 是 | 位编码（如 03），最长16字符 |
| locationType | String | 是 | 库位类型：STORAGE/PICKING/STAGING/QC |
| capacity | Integer | 否 | 容量 |
| remark | String | 否 | 备注 |

**响应结构**：`R<LocationVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 库位ID |
| data.warehouseId | Long | 仓库ID |
| data.zoneCode | String | 库区编码 |
| data.rackCode | String | 货架编码 |
| data.rowCode | String | 层编码 |
| data.binCode | String | 位编码 |
| data.fullCode | String | 完整编码（自动拼接，如 WH01-A-01-02-03） |
| data.locationType | String | 库位类型 |
| data.capacity | Integer | 容量 |
| data.usedCapacity | Integer | 已用容量（默认0） |
| data.status | String | 状态：ACTIVE/INACTIVE/FROZEN（新建默认 ACTIVE） |
| data.remark | String | 备注 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- fullCode 自动拼接：仓库编码-库区-货架-层-位
- 同一仓库内 fullCode 不可重复
- 已停用的仓库不允许新增库位
- 新建默认 ACTIVE，已用容量默认 0

### 2.2 更新库位

**接口路径**：`POST /master/warehouse/location/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| locationId | Long | 是 | Query参数，库位ID |
| capacity | Integer | 否 | 容量 |
| remark | String | 否 | 备注 |

**响应结构**：`R<LocationVO>`

**业务规则**：
- 四级编码、类型、仓库ID 不可修改
- 冻结状态的库位不允许修改

### 2.3 冻结库位

**接口路径**：`POST /master/warehouse/location/freeze`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| locationId | Long | 是 | Query参数，库位ID |

**响应结构**：`R<Void>`

**业务规则**：
- 冻结后不可进行出入库操作
- 已冻结的库位重复冻结返回错误

### 2.4 解冻库位

**接口路径**：`POST /master/warehouse/location/unfreeze`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| locationId | Long | 是 | Query参数，库位ID |

**响应结构**：`R<Void>`

**业务规则**：
- 未冻结的库位不允许解冻

### 2.5 停用库位

**接口路径**：`POST /master/warehouse/location/deactivate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| locationId | Long | 是 | Query参数，库位ID |

**响应结构**：`R<Void>`

**业务规则**：
- 冻结状态的库位不允许停用，需先解冻

### 2.6 删除库位

**接口路径**：`POST /master/warehouse/location/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| locationId | Long | 是 | Query参数，库位ID |

**响应结构**：`R<Void>`

**业务规则**：
- 冻结状态的库位不允许删除

## 3. 相关错误码

| 错误码 | 说明 |
|--------|------|
| 20013 | 库位已冻结，不可进行出入库操作（LOCATION_FROZEN） |
| 10008 | 数据已存在（仓库编码重复、库位编码重复） |
| 10009 | 数据不存在 |
| 10010 | 当前状态不允许此操作 |
| 10011 | 并发冲突，数据已被他人修改 |
