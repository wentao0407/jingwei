# 客户管理接口文档

模块路径：`com.jingwei.master`  
Controller：`CustomerController`

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串，如 `?customerId=1`）；JSON Body = `@RequestBody`（请求体 JSON）  
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

---

## 1. 创建客户

**接口路径**：`POST /master/customer/create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 是 | 客户名称，最长128字符，全局唯一 |
| shortName | String | 否 | 简称，最长64字符 |
| type | String | 是 | 客户类型：WHOLESALE/RETAIL/ONLINE/FRANCHISE |
| level | String | 否 | 客户等级：A/B/C/D（默认 C） |
| contactPerson | String | 否 | 联系人，最长32字符 |
| contactPhone | String | 否 | 联系电话，最长20字符 |
| address | String | 否 | 地址 |
| deliveryAddress | String | 否 | 默认发货地址 |
| settlementType | String | 否 | 结算方式：MONTHLY/QUARTERLY/COD |
| creditLimit | BigDecimal | 否 | 信用额度（参考值，本期不做超额拦截） |
| remark | String | 否 | 备注 |

**响应结构**：`R<CustomerVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 客户ID |
| data.code | String | 客户编码（自动生成，格式 CUS-000001） |
| data.name | String | 客户名称 |
| data.shortName | String | 简称 |
| data.type | String | 客户类型：WHOLESALE/RETAIL/ONLINE/FRANCHISE |
| data.level | String | 客户等级：A/B/C/D |
| data.contactPerson | String | 联系人 |
| data.contactPhone | String | 联系电话 |
| data.address | String | 地址 |
| data.deliveryAddress | String | 默认发货地址 |
| data.settlementType | String | 结算方式：MONTHLY/QUARTERLY/COD |
| data.creditLimit | BigDecimal | 信用额度 |
| data.status | String | 状态：ACTIVE/INACTIVE（新建默认 ACTIVE） |
| data.remark | String | 备注 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 客户编码由编码规则引擎自动生成（格式：CUS-6位流水号），不可手动指定
- 客户名称全局唯一
- 创建后默认状态为 ACTIVE，等级默认 C
- 信用额度为参考值，本期不做超额拦截

## 2. 更新客户

**接口路径**：`POST /master/customer/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customerId | Long | 是 | Query参数，客户ID |
| name | String | 否 | 客户名称，最长128字符（修改时校验唯一性） |
| shortName | String | 否 | 简称，最长64字符 |
| level | String | 否 | 客户等级：A/B/C/D |
| contactPerson | String | 否 | 联系人，最长32字符 |
| contactPhone | String | 否 | 联系电话，最长20字符 |
| address | String | 否 | 地址 |
| deliveryAddress | String | 否 | 默认发货地址 |
| settlementType | String | 否 | 结算方式：MONTHLY/QUARTERLY/COD |
| creditLimit | BigDecimal | 否 | 信用额度 |
| remark | String | 否 | 备注 |

**响应结构**：`R<CustomerVO>`

**业务规则**：
- 编码（code）和类型（type）不可修改——编码被业务单据引用，类型影响定价策略和销售渠道
- 名称修改时校验全局唯一性（排除自身）
- 所有字段可选，只更新传入的字段

---

## 3. 停用客户

**接口路径**：`POST /master/customer/deactivate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customerId | Long | 是 | Query参数，客户ID |

**响应结构**：`R<Void>`

**业务规则**：
- 停用后不可创建新销售订单，已有订单不受影响
- 已停用的客户重复停用返回错误

---

## 4. 启用客户

**接口路径**：`POST /master/customer/activate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customerId | Long | 是 | Query参数，客户ID |

**响应结构**：`R<Void>`

**业务规则**：
- 将停用的客户重新启用，启用后可创建新销售订单
- 已启用的客户重复启用返回错误

---

## 5. 删除客户

**接口路径**：`POST /master/customer/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customerId | Long | 是 | Query参数，客户ID |

**响应结构**：`R<Void>`

**业务规则**：
- 仅允许删除未被销售订单引用的客户
- 已被引用时返回错误提示（当前订单模块未实现，预留钩子）
- 逻辑删除，不物理删除

## 7. 分页查询客户

**接口路径**：`POST /master/customer/page`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | Long | 否 | 当前页码，默认1 |
| size | Long | 否 | 每页条数，默认10 |
| type | String | 否 | 客户类型筛选：WHOLESALE/RETAIL/ONLINE/FRANCHISE |
| level | String | 否 | 客户等级筛选：A/B/C/D |
| status | String | 否 | 状态筛选：ACTIVE/INACTIVE |
| keyword | String | 否 | 关键词搜索（匹配编码或名称） |

**响应结构**：`R<IPage<CustomerVO>>`

分页结构包含 records（数据列表）、total（总条数）、current（当前页码）、size（每页条数）等字段。

## 8. 客户相关错误码

| 错误码 | 说明 |
|--------|------|
| 20012 | 客户名称已存在（CUSTOMER_NAME_DUPLICATE） |
| 10008 | 数据已存在（编码重复等通用场景） |
| 10009 | 数据不存在 |
| 10010 | 当前状态不允许此操作（如已停用再停用） |
| 10011 | 并发冲突，数据已被他人修改 |
