# 季节与波段管理接口文档

模块路径：`com.jingwei.master`（季节部分）  
Controller：`SeasonController`

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串，如 `?seasonId=1`）；JSON Body = `@RequestBody`（请求体 JSON）  
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

---

## 1. 季节接口

### 1.1 创建季节

**接口路径**：`POST /master/season/create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | String | 是 | 季节编码，最长16字符，全局唯一（如 2026SS） |
| name | String | 是 | 季节名称，最长64字符（如 2026春夏） |
| year | Integer | 是 | 年份（如 2026） |
| seasonType | String | 是 | 季节类型：SPRING_SUMMER/AUTUMN_WINTER |
| startDate | LocalDate | 是 | 开始日期 |
| endDate | LocalDate | 是 | 结束日期 |

**响应结构**：`R<SeasonVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 季节ID |
| data.code | String | 季节编码 |
| data.name | String | 季节名称 |
| data.year | Integer | 年份 |
| data.seasonType | String | 季节类型：SPRING_SUMMER/AUTUMN_WINTER |
| data.startDate | LocalDate | 开始日期 |
| data.endDate | LocalDate | 结束日期 |
| data.status | String | 状态：ACTIVE/CLOSED |
| data.waves | List\<WaveVO\> | 波段列表（创建时为空） |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 编码全局唯一
- 创建后默认状态为 ACTIVE
- 同一年份+季节类型组合应唯一

### 1.2 更新季节

**接口路径**：`POST /master/season/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| seasonId | Long | 是 | Query参数，季节ID |
| name | String | 否 | 季节名称，最长64字符 |
| startDate | LocalDate | 否 | 开始日期 |
| endDate | LocalDate | 否 | 结束日期 |

**响应结构**：`R<SeasonVO>`

**业务规则**：
- 编码、年份、季节类型不可修改
- 已关闭的季节不允许更新
- 所有字段可选，只更新传入的字段

### 1.3 关闭季节

**接口路径**：`POST /master/season/close`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| seasonId | Long | 是 | Query参数，季节ID |

**响应结构**：`R<Void>`

**业务规则**：
- 关闭后不可在业务单据中选择，其下波段也不可选用
- 关闭是单向操作，不可恢复
- 已关闭的季节重复关闭返回错误

### 1.4 删除季节

**接口路径**：`POST /master/season/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| seasonId | Long | 是 | Query参数，季节ID |

**响应结构**：`R<Void>`

**业务规则**：
- 删除时同时删除季节下的所有波段
- 已被业务引用的季节不可删除（当前预留钩子，模块未实现时允许删除）

### 1.5 查询季节列表

**接口路径**：`POST /master/season/list`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| year | Integer | 否 | 年份筛选 |
| seasonType | String | 否 | 季节类型筛选：SPRING_SUMMER/AUTUMN_WINTER |
| status | String | 否 | 状态筛选：ACTIVE/CLOSED |

**响应结构**：`R<List<SeasonVO>>`

列表结果不含波段详情（waves 为 null），需通过详情接口获取。

### 1.6 查询季节详情

**接口路径**：`POST /master/season/detail`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| seasonId | Long | 是 | Query参数，季节ID |

**响应结构**：`R<SeasonVO>`

详情包含波段列表（waves）。

## 2. 波段接口

### 2.1 新增波段

**接口路径**：`POST /master/season/wave/create`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| seasonId | Long | 是 | Query参数，季节ID |
| code | String | 是 | 波段编码，最长16字符，组内唯一（如 2026SS-W1） |
| name | String | 是 | 波段名称，最长64字符（如 春一） |
| deliveryDate | LocalDate | 否 | 交货日期 |
| sortOrder | Integer | 否 | 排序号，不传则追加到末尾 |

**响应结构**：`R<WaveVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 波段ID |
| data.seasonId | Long | 所属季节ID |
| data.code | String | 波段编码 |
| data.name | String | 波段名称 |
| data.deliveryDate | LocalDate | 交货日期 |
| data.sortOrder | Integer | 排序号 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 已关闭的季节不允许新增波段

### 2.2 更新波段

**接口路径**：`POST /master/season/wave/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| waveId | Long | 是 | Query参数，波段ID |
| name | String | 否 | 波段名称，最长64字符 |
| deliveryDate | LocalDate | 否 | 交货日期 |
| sortOrder | Integer | 否 | 排序号 |

**响应结构**：`R<WaveVO>`

**业务规则**：
- 已关闭的季节下的波段不允许修改
- 波段编码不可修改
- 所有字段可选，只更新传入的字段

### 2.3 删除波段

**接口路径**：`POST /master/season/wave/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| waveId | Long | 是 | Query参数，波段ID |

**响应结构**：`R<Void>`

**业务规则**：
- 已关闭的季节下的波段不允许删除
