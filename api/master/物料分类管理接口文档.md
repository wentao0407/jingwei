# 物料分类管理接口文档

模块路径：`com.jingwei.master`（分类部分）  
Controller：`CategoryController`

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串，如 `?categoryId=1`）；JSON Body = `@RequestBody`（请求体 JSON）  
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

---

## 1. 创建物料分类

**接口路径**：`POST /master/category/create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| parentId | Long | 否 | 父级分类ID，不传或null表示顶级分类 |
| code | String | 是 | 分类编码，最长32字符，同级唯一 |
| name | String | 是 | 分类名称，最长64字符 |
| sortOrder | Integer | 否 | 排序号，默认0 |

**响应结构**：`R<CategoryTreeVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 分类ID |
| data.parentId | Long | 父级分类ID |
| data.code | String | 分类编码 |
| data.name | String | 分类名称 |
| data.level | Integer | 层级（1/2/3），由后端自动计算 |
| data.sortOrder | Integer | 排序号 |
| data.status | String | 状态：ACTIVE/INACTIVE |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |
| data.children | List\<CategoryTreeVO\> | 子分类列表 |

**业务规则**：
- 分类最多3级，第4级创建时返回错误码 20001
- 同一父级下编码不可重复
- 父级分类不存在或已停用时不可创建子分类
- 层级由后端根据 parentId 自动计算，无需前端传入

---

## 2. 更新物料分类

**接口路径**：`POST /master/category/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| categoryId | Long | 是 | Query参数，分类ID |
| code | String | 否 | 分类编码，最长32字符 |
| name | String | 否 | 分类名称，最长64字符 |
| sortOrder | Integer | 否 | 排序号 |
| status | String | 否 | 状态：ACTIVE/INACTIVE |

**响应结构**：`R<CategoryTreeVO>`

**业务规则**：
- 所有字段可选，只更新传入的字段
- 不允许变更 parentId 和 level（如需移动分类，应删除后重建）
- 修改编码时校验同级唯一性

---

## 3. 删除物料分类

**接口路径**：`POST /master/category/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| categoryId | Long | 是 | Query参数，分类ID |

**响应结构**：`R<Void>`

**业务规则**：
- 存在子分类时不可删除，返回错误码 20002
- 被物料引用时不可删除，返回错误码 20003，提示引用物料数量
- 删除为逻辑删除（BaseEntity 的 @TableLogic deleted 字段），数据不会从数据库中物理移除

---

## 4. 查询分类树

**接口路径**：`POST /master/category/tree`

**请求参数**：无

**响应结构**：`R<List<CategoryTreeVO>>`

返回完整的树形结构，顶级节点包含嵌套的子节点，最多3级。

---

## 5. 根据ID查询分类详情

**接口路径**：`POST /master/category/detail`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| categoryId | Long | 是 | Query参数，分类ID |

**响应结构**：`R<CategoryTreeVO>`

返回该分类节点信息，`children` 为该节点的直接子分类列表。
