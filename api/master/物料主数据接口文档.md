# 物料主数据接口文档

模块路径：`com.jingwei.master`（物料部分）  
Controller：`MaterialController`

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串，如 `?materialId=1`）；JSON Body = `@RequestBody`（请求体 JSON）  
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

> **已实现**：物料 CRUD、分页查询、属性定义查询  
> **待实现**：属性定义 CRUD（当前只有按物料类型查询属性定义，不支持新增/修改/删除属性定义）

---

## 1. 创建物料

**接口路径**：`POST /master/material/create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 是 | 物料名称，最长128字符 |
| type | String | 是 | 物料类型：FABRIC/TRIM/PACKAGING |
| categoryId | Long | 否 | 物料分类ID |
| unit | String | 是 | 基本单位（如 米/个/套/件） |
| extAttrs | Map\<String, Object\> | 否 | 扩展属性（按 type 不同存不同结构） |
| remark | String | 否 | 备注 |

**响应结构**：`R<MaterialVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 物料ID |
| data.code | String | 物料编码（后端自动生成） |
| data.name | String | 物料名称 |
| data.type | String | 物料类型：FABRIC/TRIM/PACKAGING |
| data.categoryId | Long | 物料分类ID |
| data.unit | String | 基本单位 |
| data.status | String | 状态：ACTIVE/INACTIVE |
| data.extAttrs | Map\<String, Object\> | 扩展属性 |
| data.remark | String | 备注 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 物料编码由后端编码规则引擎自动生成，前端无需传入
- extAttrs 中必填属性由属性定义（AttributeDef）决定，缺失时返回具体缺哪些字段
- 面料（FABRIC）类型如有 composition 字段，成分百分比合计必须等于100%
- 创建后默认状态为 ACTIVE

## 2. 更新物料

**接口路径**：`POST /master/material/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| materialId | Long | 是 | Query参数，物料ID |
| name | String | 否 | 物料名称 |
| categoryId | Long | 否 | 物料分类ID |
| unit | String | 否 | 基本单位 |
| extAttrs | Map\<String, Object\> | 否 | 扩展属性，传入则整体替换 |
| remark | String | 否 | 备注 |
| status | String | 否 | 状态：ACTIVE/INACTIVE |

**响应结构**：`R<MaterialVO>`

**业务规则**：
- 物料编码和类型不可修改
- 所有字段可选，只更新传入的字段

---

## 3. 停用物料

**接口路径**：`POST /master/material/deactivate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| materialId | Long | 是 | Query参数，物料ID |

**响应结构**：`R<Void>`

**业务规则**：
- 停用后不可在业务单据中选择
- 已停用的物料再次停用返回错误

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| materialId | Long | 是 | Query参数，物料ID |

**响应结构**：`R<MaterialVO>`

## 5. 分页查询物料

**接口路径**：`POST /master/material/page`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | Long | 否 | 页码，默认1 |
| size | Long | 否 | 每页条数，默认20 |
| type | String | 否 | 物料类型筛选：FABRIC/TRIM/PACKAGING |
| categoryId | Long | 否 | 物料分类ID筛选 |
| status | String | 否 | 状态筛选：ACTIVE/INACTIVE |
| keyword | String | 否 | 关键词搜索（编码或名称模糊匹配） |

**响应结构**：`R<IPage<MaterialVO>>`

分页结构见 [全局错误码与通用结构说明](../全局错误码说明.md)，`records` 为 `MaterialVO` 列表。

---

## 6. 查询指定物料类型的属性定义

**接口路径**：`POST /master/material/attributeDefs`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| materialType | String | 是 | Query参数，物料类型：FABRIC/TRIM/PACKAGING |

**响应结构**：`R<List<AttributeDefVO>>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data[].id | Long | 属性定义ID |
| data[].code | String | 属性编码 |
| data[].name | String | 属性名称 |
| data[].materialType | String | 适用物料类型 |
| data[].inputType | String | 输入类型：TEXT/NUMBER/SELECT/MULTI_SELECT/COMPOSITION |
| data[].required | Boolean | 是否必填 |
| data[].sortOrder | Integer | 排序号 |
| data[].options | List\<String\> | 选项列表（SELECT 类型用） |
| data[].extJsonPath | String | JSONB 中对应路径 |

**业务规则**：
- 前端根据此接口渲染动态属性表单
- 不同物料类型返回不同的属性定义集合
