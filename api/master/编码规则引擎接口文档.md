# 编码规则引擎接口文档

模块路径：`com.jingwei.master`（编码规则部分）  
Controller：`CodingRuleController`

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串，如 `?ruleId=1`）；JSON Body = `@RequestBody`（请求体 JSON）  
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

---

## 1. 创建编码规则

**接口路径**：`POST /master/codingRule/create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | String | 是 | 规则编码（全局唯一） |
| name | String | 是 | 规则名称 |
| businessType | String | 否 | 业务类型 |
| description | String | 否 | 说明 |
| segments | List\<CodingRuleSegmentDTO\> | 是 | 段列表，不能为空 |

**CodingRuleSegmentDTO**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| segmentType | String | 是 | 段类型：FIXED/DATE/SEQUENCE/SEASON/WAREHOUSE/CUSTOM |
| segmentValue | String | 否 | 段值（FIXED 类型为固定文本，DATE 类型为格式如 yyyyMM） |
| seqLength | Integer | 否 | 流水号长度（SEQUENCE 类型专用） |
| seqResetType | String | 否 | 流水号重置方式（SEQUENCE 类型专用）：NEVER/YEARLY/MONTHLY/DAILY |
| connector | String | 否 | 连接符 |
| sortOrder | Integer | 是 | 排序号 |

**响应结构**：`R<CodingRuleVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 规则ID |
| data.code | String | 规则编码 |
| data.name | String | 规则名称 |
| data.businessType | String | 业务类型 |
| data.description | String | 说明 |
| data.status | String | 状态：ACTIVE/INACTIVE |
| data.used | Boolean | 是否已被使用（已生成过编码） |
| data.segments | List\<CodingRuleSegmentVO\> | 段列表 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 规则编码全局唯一
- 段列表至少包含一个 SEQUENCE 类型的段

---

## 2. 更新编码规则

**接口路径**：`POST /master/codingRule/update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ruleId | Long | 是 | Query参数，规则ID |
| name | String | 否 | 规则名称 |
| businessType | String | 否 | 业务类型 |
| description | String | 否 | 说明 |
| status | String | 否 | 状态：ACTIVE/INACTIVE |

**响应结构**：`R<CodingRuleVO>`

**业务规则**：
- 规则编码不可修改
- 段列表不可通过更新接口修改（需重建规则）
- 已使用的规则不可删除，但可停用

---

## 3. 删除编码规则

**接口路径**：`POST /master/codingRule/delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ruleId | Long | 是 | Query参数，规则ID |

**响应结构**：`R<Void>`

**业务规则**：
- 已使用（已生成过编码）的规则不可删除

---

## 4. 查询所有编码规则

**接口路径**：`POST /master/codingRule/list`

**请求参数**：无

**响应结构**：`R<List<CodingRuleVO>>`

返回所有编码规则，每个规则包含段列表。

---

## 5. 根据编码查询规则详情

**接口路径**：`POST /master/codingRule/detail`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | String | 是 | Query参数，规则编码 |

**响应结构**：`R<CodingRuleVO>`

---

## 6. 生成编码

**接口路径**：`POST /master/codingRule/generate`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ruleCode | String | 是 | 规则编码 |
| context | Map\<String, String\> | 否 | 上下文变量（如仓库编码、季节编码、自定义值等） |

**响应结构**：`R<String>`

返回生成的编码字符串。

**业务规则**：
- 流水号原子递增，使用数据库行级锁保证并发安全
- DATE 段根据当前日期自动填充
- SEASON/WAREHOUSE/CUSTOM 段从 context 中取值
- 规则不存在或已停用时返回错误

---

## 7. 预览编码

**接口路径**：`POST /master/codingRule/preview`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ruleCode | String | 是 | 规则编码 |
| context | Map\<String, String\> | 否 | 上下文变量 |

**响应结构**：`R<CodePreviewVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.previewCode | String | 预览编码（不递增流水号，使用示例值 00001） |

**业务规则**：
- 预览不会递增流水号，仅用于前端展示效果
- 流水号部分显示为示例值（如 00001）
