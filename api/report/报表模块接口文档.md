# 报表模块接口文档

## 1. 库存台账

### 1.1 分页查询库存台账

- **接口路径**: `/report/ledger/page`
- **请求方法**: POST
- **权限标识**: `report:ledger:view`

**请求参数**:

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| current | Long | 否 | 当前页，默认 1 |
| size | Long | 否 | 每页大小，默认 20 |
| inventoryType | String | 否 | 库存类型：SKU/MATERIAL，默认查询成品 |
| warehouseId | Long | 否 | 仓库ID |
| categoryId | Long | 否 | 品类ID（仅成品有效） |
| seasonId | Long | 否 | 季节ID（仅成品有效） |
| skuId | Long | 否 | SKU ID（精确查询，仅成品有效） |
| materialId | Long | 否 | 物料ID（精确查询，仅原料有效） |
| keyword | String | 否 | SKU编码/物料编码/名称模糊搜索 |

**响应结构**:

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "records": [
      {
        "inventoryId": 1,
        "inventoryType": "SKU",
        "skuId": 1,
        "skuCode": "SP20260001-BK-M",
        "materialId": null,
        "materialCode": null,
        "materialName": null,
        "spuId": 1,
        "spuCode": "SP20260001",
        "spuName": "春季连衣裙",
        "colorWayId": 1,
        "colorName": "黑色",
        "sizeId": 1,
        "sizeCode": "M",
        "warehouseId": 1,
        "warehouseName": "主仓库",
        "batchNo": "B001",
        "availableQty": 80,
        "lockedQty": 10,
        "qcQty": 5,
        "totalQty": 95,
        "unitCost": 50.00,
        "totalAmount": 4750.00,
        "lastInboundDate": "2026-05-01",
        "lastOutboundDate": null,
        "updatedAt": "2026-05-01T10:00:00"
      }
    ],
    "total": 100,
    "size": 20,
    "current": 1
  }
}
```

**业务规则**:
- inventoryType=SKU 时查询成品库存（关联 SKU、SPU、颜色款、尺码）
- inventoryType=MATERIAL 时查询原料库存（关联物料主数据）
- 不指定 inventoryType 时默认查询成品库存
- categoryId 和 seasonId 仅对成品查询有效
- totalAmount = totalQty × unitCost

---

### 1.2 查询库存台账矩阵视图（颜色 × 尺码）

- **接口路径**: `/report/ledger/matrix`
- **请求方法**: POST
- **权限标识**: `report:ledger:view`

**请求参数**:

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| spuId | Long | 是 | 款式ID |
| warehouseId | Long | 是 | 仓库ID |

**响应结构**:

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "spuId": 1,
    "warehouseId": 1,
    "sizes": ["S", "M", "L", "XL"],
    "matrix": {
      "黑色": {"S": 10, "M": 20, "L": 15, "XL": 5},
      "白色": {"S": 5, "M": 8, "L": 12, "XL": 3}
    },
    "colorTotals": {"黑色": 50, "白色": 28},
    "sizeTotals": {"S": 15, "M": 28, "L": 27, "XL": 8},
    "grandTotal": 78
  }
}
```

**业务规则**:
- 指定款式和仓库，返回该款式下所有 SKU 的库存数量矩阵
- 行为颜色（按 ColorWay.sort_order 排序），列为尺码（按 Size.sort_order 排序）
- 无库存数据时返回 null
- 汇总包括：颜色行合计、尺码列合计、总合计

---

### 1.3 导出库存台账

- **接口路径**: `/report/ledger/export`
- **请求方法**: POST
- **权限标识**: `report:ledger:export`

**请求参数**: 同 1.1

**响应**: Excel 文件流（.xlsx）

**成品表头**: SKU编码、款式编码、款式名称、颜色、尺码、仓库、批次号、可用数量、锁定数量、质检数量、实际库存、单位成本、库存金额、最后入库日期、最后出库日期

**原料表头**: 物料编码、物料名称、仓库、批次号、可用数量、锁定数量、质检数量、实际库存、单位成本、库存金额、最后入库日期、最后出库日期

---

## 2. 出入库流水

### 2.1 分页查询出入库流水

- **接口路径**: `/report/flow/page`
- **请求方法**: POST
- **权限标识**: `report:flow:view`

**请求参数**:

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| current | Long | 否 | 当前页，默认 1 |
| size | Long | 否 | 每页大小，默认 20 |
| inventoryType | String | 否 | 库存类型：SKU/MATERIAL |
| operationType | String | 否 | 操作类型（见 OperationType 枚举） |
| warehouseId | Long | 否 | 仓库ID |
| skuId | Long | 否 | SKU ID |
| materialId | Long | 否 | 物料ID |
| sourceType | String | 否 | 来源单据类型 |
| startTime | DateTime | 否 | 开始时间 |
| endTime | DateTime | 否 | 结束时间 |
| operationNo | String | 否 | 操作单号模糊搜索 |

**响应结构**:

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "records": [
      {
        "id": 1,
        "operationNo": "OP20260501001",
        "operationType": "INBOUND_PURCHASE",
        "operationTypeLabel": "采购到货",
        "inventoryType": "SKU",
        "skuCode": "SP20260001-BK-M",
        "materialCode": null,
        "materialName": null,
        "warehouseName": "主仓库",
        "batchNo": "B001",
        "quantity": 100,
        "totalBefore": 0,
        "totalAfter": 100,
        "changeQty": 100,
        "unitCost": 50.00,
        "costAmount": 5000.00,
        "sourceType": "ASN",
        "sourceNo": "ASN001",
        "operatorId": 1,
        "operatedAt": "2026-05-01T10:00:00",
        "remark": "到货入库"
      }
    ],
    "total": 500,
    "size": 20,
    "current": 1
  }
}
```

**业务规则**:
- changeQty 表示实际库存变动：入库类操作为正数，出库类操作为负数
- 操作类型：INBOUND_PURCHASE(采购到货)、INBOUND_PRODUCTION(生产入库)、INBOUND_RETURN(退货入库)、QC_PASS(质检合格)、QC_FAIL(质检不合格)、ALLOCATE(锁定预留)、RELEASE(释放预留)、OUTBOUND_SALES(销售出库)、OUTBOUND_MATERIAL(领料出库)、ADJUST_GAIN(盘盈调增)、ADJUST_LOSS(盘亏调减)
- ALLOCATE 和 RELEASE 不改变总库存（changeQty = 0）

---

### 2.2 导出出入库流水

- **接口路径**: `/report/flow/export`
- **请求方法**: POST
- **权限标识**: `report:flow:export`

**请求参数**: 同 2.1

**响应**: Excel 文件流（.xlsx）

**表头**: 操作单号、操作类型、库存类型、SKU/物料编码、物料名称、仓库、批次号、操作数量、变动数量、操作前库存、操作后库存、单位成本、成本金额、来源单据、来源编号、操作时间、备注

---

## 4. 畅滞销分析

### 4.1 查询畅滞销分析

- **接口路径**: `/report/turnover/page`
- **请求方法**: POST
- **权限标识**: `report:turnover:view`

**请求参数**:

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| current | Long | 否 | 当前页，默认 1 |
| size | Long | 否 | 每页大小，默认 20 |
| inventoryType | String | 否 | 库存类型：SKU/MATERIAL |
| warehouseId | Long | 否 | 仓库ID |
| categoryId | Long | 否 | 品类ID（仅成品有效） |
| seasonId | Long | 否 | 季节ID（仅成品有效） |
| startDate | Date | 否 | 统计开始日期，默认30天前 |
| endDate | Date | 否 | 统计结束日期，默认今天 |
| keyword | String | 否 | 关键字搜索 |

**响应结构**:

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "records": [
      {
        "skuCode": "SP20260001-BK-M",
        "spuCode": "SP20260001",
        "spuName": "春季连衣裙",
        "colorName": "黑色",
        "sizeCode": "M",
        "warehouseName": "主仓库",
        "currentQty": 100,
        "outboundQty": 500,
        "inboundQty": 50,
        "netOutboundQty": 450,
        "avgInventory": 325,
        "turnoverDays": 19.5,
        "turnoverRate": 1.54,
        "turnoverGrade": "FAST",
        "turnoverGradeLabel": "畅销",
        "startDate": "2026-04-05",
        "endDate": "2026-05-05",
        "periodDays": 30
      }
    ],
    "total": 100,
    "size": 20,
    "current": 1
  }
}
```

**业务规则**:
- 期初库存 ≈ 当前库存 + 净出库（简化估算）
- 平均库存 = (期初库存 + 期末库存) / 2
- 周转天数 = 统计天数 × 平均库存 / 出库数量
- 周转率 = 出库数量 / 平均库存
- 畅销等级分类：
  - FAST（畅销）：周转天数 ≤ 15天
  - NORMAL（正常）：周转天数 16-60天
  - SLOW（滞销）：周转天数 61-90天
  - DEAD（呆滞）：周转天数 > 90天或无出库记录
- 按出库数量降序排列（最畅销的排在前面）

---

- **接口路径**: `/report/turnover/export`
- **请求方法**: POST
- **权限标识**: `report:turnover:export`

**请求参数**: 同 4.1

**响应**: Excel 文件流（.xlsx）

**表头**: SKU/物料编码、款式编码、款式名称、颜色、尺码、仓库、当前库存、出库数量、入库数量、净出库、平均库存、周转天数、周转率、畅销等级
