# 库存管理核心接口文档

> 模块：inventory（库存管理）
> 任务编号：T-29 / T-30
> 更新日期：2026-05-05
> 说明：T-29/T-30 为领域层服务，不直接暴露 REST 接口。库存变更通过其他模块的业务接口间接调用（如采购到货、销售出库等）。本文档记录核心服务的方法签名和业务规则。

## 一、InventoryDomainService — 核心库存变更服务

### changeInventory(ChangeInventoryCommand)

所有库存变更的唯一入口，禁止直接 UPDATE 数量字段。

#### 方法签名

```java
public void changeInventory(ChangeInventoryCommand cmd)
```

#### ChangeInventoryCommand 参数

| 字段名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| operationType | OperationType | 是 | 操作类型（见下方枚举） |
| inventoryType | InventoryType | 是 | SKU 或 MATERIAL |
| inventoryId | Long | 是 | 库存记录ID |
| skuId | Long | 成品时必填 | SKU ID |
| materialId | Long | 原料时必填 | 物料ID |
| warehouseId | Long | 是 | 仓库ID |
| batchNo | String | 否 | 批次号 |
| quantity | BigDecimal | 是 | 操作数量（正数） |
| sourceType | String | 否 | 来源单据类型 |
| sourceId | Long | 否 | 来源单据ID |
| unitCost | BigDecimal | 否 | 单位成本（入库时使用） |
| operatorId | Long | 否 | 操作人ID |

#### 操作类型与字段变更规则

| 操作类型 | available | locked | qc | total | 说明 |
|---------|-----------|--------|-----|-------|------|
| INBOUND_PURCHASE | - | - | +N | +N | 采购到货→质检 |
| INBOUND_PRODUCTION | +N | - | - | +N | 生产完工→可用 |
| INBOUND_RETURN | - | - | +N | +N | 退货→质检 |
| QC_PASS | +N | - | -N | 不变 | 质检合格→可用 |
| QC_FAIL | - | - | -N | -N | 质检不合格→扣减 |
| ALLOCATE | -N | +N | - | 不变 | 锁定预留 |
| RELEASE | +N | -N | - | 不变 | 释放预留 |
| OUTBOUND_SALES | - | -N | - | -N | 销售出库 |
| OUTBOUND_MATERIAL | -N | - | - | -N | 领料出库 |
| ADJUST_GAIN | +N | - | - | +N | 盘盈 |
| ADJUST_LOSS | -N | - | - | -N | 盘亏 |

#### 校验规则

- ALLOCATE / OUTBOUND_MATERIAL / ADJUST_LOSS：available < quantity → 抛 INSUFFICIENT_INVENTORY
- OUTBOUND_SALES / RELEASE：locked < quantity → 抛 LOCKED_INVENTORY_INSUFFICIENT
- QC_PASS / QC_FAIL：qc < quantity → 抛 QC_INVENTORY_INSUFFICIENT
- 库存记录不存在 → 抛 INVENTORY_NOT_FOUND
- 乐观锁冲突（3次重试后） → 抛 CONCURRENT_CONFLICT

#### 并发安全

乐观锁（version 字段）+ 最多重试3次 + 退避 50~150ms 随机等待。

## 二、AllocationDomainService — 库存预留服务

### allocate — 库存预留

销售订单确认时调用，为订单行预留可用库存。

```java
public AllocationResult allocate(Long skuId, Long warehouseId, int quantity,
                                  String orderType, Long orderId, Long orderLineId,
                                  Long operatorId)
```

**返回 AllocationResult：**

| 字段 | 类型 | 说明 |
|------|------|------|
| allocatedQty | int | 实际预留量 |
| shortfall | int | 缺口数量（需求量 - 预留量） |
| allocationId | Long | 预留记录ID（无库存时为null） |

**业务规则：**
- 全额预留：available >= 需求量 → 全部锁定
- 部分预留：available < 需求量 → 锁定可用部分，返回缺口
- 无库存：available = 0 → 不预留，返回全部缺口
- 预留过期天数从系统配置 `inventory.allocation.expiry.days` 读取，默认7天

### release — 释放预留

订单取消时调用，将锁定库存归还到可用库存。

```java
public void release(Long allocationId, Long operatorId)
```

**业务规则：**
- 只有 ACTIVE 或 PARTIAL_FULFILLED 状态允许释放
- 释放后状态变为 RELEASED，remaining_qty 清零

### fulfill — 出库完成

出库操作完成时调用，更新预留记录的已出库量。

```java
public void fulfill(Long allocationId, int outboundQty, Long operatorId)
```

**业务规则：**
- fulfilled_qty += outboundQty
- 全部出库 → status = FULFILLED
- 部分出库 → status = PARTIAL_FULFILLED

### releaseExpired — 过期自动释放

定时任务调用，扫描并释放所有过期的 ACTIVE 预留记录。

```java
public int releaseExpired(Long operatorId)
```

## 三、数据库表

| 表名 | 说明 | 唯一约束 |
|------|------|---------|
| t_inventory_sku | 成品库存 | (sku_id, warehouse_id, batch_no) |
| t_inventory_material | 原料库存 | (material_id, warehouse_id, batch_no) |
| t_inventory_in_transit | 在途库存 | - |
| t_inventory_operation | 操作记录 | - |

## 四、错误码

| 错误码 | 含义 |
|--------|------|
| 50001 | 库存不足 |
| 50002 | 库存预留已过期 |
| 50003 | 盘点进行中 |
| 50004 | 锁定库存不足 |
| 50005 | 质检库存不足 |
| 50006 | 库存记录不存在 |
| 50007 | 预留记录不存在 |
| 50008 | 预留状态不允许此操作 |
