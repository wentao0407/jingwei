# 发运管理接口文档

Controller：`ShipmentController`

> 说明：当前发运单以出库单作为聚合根，发运分页和详情返回 `OutboundOrderVO` 结构；确认发运会扣减库存并按销售订单状态机推进销售订单状态。

## 1. 分页查询发运单

**接口路径**：`POST /warehouse/shipment/page`

**请求体**：`ShipmentQueryDTO`

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| current | Long | 否 | 当前页，默认 1 |
| size | Long | 否 | 每页大小，默认 20 |
| status | String | 否 | 出库/发运状态：DRAFT/CONFIRMED/SHIPPED/CANCELLED |
| warehouseId | Long | 否 | 仓库 ID |
| outboundNo | String | 否 | 出库单号模糊查询 |

**响应结构**：`R<IPage<OutboundOrderVO>>`

## 2. 查询发运单详情

**接口路径**：`POST /warehouse/shipment/detail`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| shipmentId | Long | 是 | 发运聚合 ID，当前等同出库单 ID |

**响应结构**：`R<OutboundOrderVO>`

详情包含 `lines`，行结构为 `OutboundOrderLineVO`。

## 3. 确认发运

**接口路径**：`POST /warehouse/shipment/confirm`

**请求体**：`ConfirmShipmentDTO`

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| outboundId | Long | 是 | 出库单 ID |
| salesOrderId | Long | 否 | 销售订单 ID；销售出库时可用于推进销售订单状态 |

**响应结构**：`R<Void>`

**业务规则**：

- 调用出库领域服务确认出库，实际扣减库存。
- 若传入销售订单 ID，则触发销售订单 `SHIP` 事件。
- 已发运或状态不允许的出库单会返回后端业务错误。
