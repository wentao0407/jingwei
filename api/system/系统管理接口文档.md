# 系统管理模块接口文档

模块路径：`com.jingwei.system`
包含：认证、用户管理、角色管理、菜单管理、数据权限、操作日志、系统配置

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串，如 `?userId=1`）；JSON Body = `@RequestBody`（请求体 JSON）
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

---

## 1. 认证

### 1.1 登录

**接口路径**：`POST /auth/login`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |

**响应结构**：`R<LoginVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0=成功 |
| data.token | String | JWT Token |
| data.userId | Long | 用户ID |
| data.username | String | 用户名 |
| data.realName | String | 真实姓名 |
| data.roleIds | List\<Long\> | 角色ID列表 |
| data.permissions | List\<String\> | 权限标识列表 |
| data.menuTree | List\<MenuVO\> | 授权菜单树（目录+菜单，不含按钮） |
| data.passwordExpired | boolean | 密码是否已过期，true时前端应引导用户修改密码 |

**业务规则**：
- 用户名或密码错误返回错误码 401
- 用户被停用时不允许登录
- 登录时检查密码是否过期（过期天数从系统配置 `password.expiry.days` 读取，默认90天，0表示永不过期）
- 密码过期时仍允许登录，但返回 `passwordExpired=true`，前端应引导用户修改密码

## 2. 用户管理

### 2.1 创建用户

**接口路径**：`POST /system/user/create`  
**权限标识**：`system:user:create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 用户名，最长50字符 |
| password | String | 是 | 密码，8-50字符，必须含大写字母、小写字母和数字 |
| realName | String | 否 | 真实姓名，最长50字符 |
| phone | String | 否 | 手机号，最长20字符 |
| email | String | 否 | 邮箱，最长100字符 |

**响应结构**：`R<UserVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 用户ID |
| data.username | String | 用户名 |
| data.realName | String | 真实姓名 |
| data.phone | String | 手机号 |
| data.email | String | 邮箱 |
| data.status | String | 状态：ACTIVE/INACTIVE |
| data.roleIds | List\<Long\> | 角色ID列表 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 用户名必须唯一，重复时返回错误码 20001
- 创建后默认状态为 ACTIVE
- 密码使用 BCrypt 加密存储
- 密码策略：至少8位，必须包含大写字母、小写字母和数字
- 创建时自动记录密码更新时间（passwordUpdatedAt），用于密码过期检查

### 2.2 更新用户

**接口路径**：`POST /system/user/update`  
**权限标识**：`system:user:update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Long | 是 | Query参数，用户ID |
| realName | String | 否 | 真实姓名，最长50字符 |
| phone | String | 否 | 手机号，最长20字符 |
| email | String | 否 | 邮箱，最长100字符 |
| status | String | 否 | 状态：ACTIVE/INACTIVE |

**响应结构**：`R<UserVO>`

**业务规则**：
- 所有字段可选，只更新传入的字段
- 用户名不可修改

### 2.3 停用用户

**接口路径**：`POST /system/user/deactivate`  
**权限标识**：`system:user:deactivate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Long | 是 | Query参数，用户ID |

**响应结构**：`R<Void>`

**业务规则**：
- 已停用的用户再次停用返回错误
- 停用后该用户无法登录

### 2.4 修改密码

**接口路径**：`POST /system/user/changePassword`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Long | 是 | Query参数，用户ID |
| oldPassword | String | 是 | 旧密码 |
| newPassword | String | 是 | 新密码，8-50字符，必须含大写字母、小写字母和数字 |

**响应结构**：`R<Void>`

**业务规则**：
- 旧密码不正确返回错误码 91014
- 新密码不能与旧密码相同，否则返回错误码 91015
- 新密码必须满足密码策略（至少8位，含大写字母、小写字母和数字）
- 修改成功后自动更新 passwordUpdatedAt，密码过期状态重置

### 2.5 分页查询用户

**接口路径**：`POST /system/user/page`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | long | 否 | 页码，默认1 |
| size | long | 否 | 每页大小，默认10 |
| keyword | String | 否 | 关键词（用户名/姓名/手机号） |
| status | String | 否 | 状态筛选 |

**响应结构**：`R<IPage<UserVO>>`

分页结构见 [全局错误码与通用结构说明](../全局错误码说明.md)，`records` 为 `UserVO` 列表。

### 2.6 查询用户详情

**接口路径**：`POST /system/user/detail`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Long | 是 | Query参数，用户ID |

**响应结构**：`R<UserVO>`

### 2.7 分配角色

**接口路径**：`POST /system/user/assignRoles`  
**权限标识**：`system:user:assignRole`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Long | 是 | Query参数，用户ID |
| roleIds | List\<Long\> | 是 | 角色ID列表，不能为空 |

**响应结构**：`R<Void>`

**业务规则**：
- 传入的 roleIds 会完全替换用户原有角色，不是追加

## 5. 数据权限

### 5.1 为角色配置数据权限

**接口路径**：`POST /system/data-scope/configure`
**权限标识**：`system:dataScope:config`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roleId | Long | 是 | Query参数，角色ID |
| scopes | List | 是 | JSON Body，权限规则列表 |

scopes 中每项：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| scopeType | String | 是 | 权限维度：WAREHOUSE/DEPT/ALL |
| scopeValue | String | 是 | 权限值：ALL=全部数据，其他为逗号分隔的ID列表 |

**响应结构**：`R<Void>`

**业务规则**：
- 全量替换：先删除该角色的所有数据权限规则，再插入新规则
- WAREHOUSE 维度的 scopeValue 为仓库ID列表（如 "1,2,3"）
- DEPT 维度的 scopeValue 为部门ID列表
- ALL 维度表示不限制，scopeValue 固定为 "ALL"

### 5.2 查询角色的数据权限

**接口路径**：`POST /system/data-scope/query`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roleId | Long | 是 | Query参数，角色ID |

**响应结构**：`R<List<DataScopeVO>>`
