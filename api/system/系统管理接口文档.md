# 系统管理模块接口文档

模块路径：`com.jingwei.system`  
包含：认证、用户管理、角色管理、菜单管理、待实现功能

**参数传递方式约定**：Query参数 = `@RequestParam`（查询字符串，如 `?userId=1`）；JSON Body = `@RequestBody`（请求体 JSON）  
**错误码说明**：见 [全局错误码说明](../全局错误码说明.md)

---

## 1. 认证

### 1.1 登录

**接口路径**：`POST /auth/login`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |

**响应结构**：`R<LoginVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0=成功 |
| data.token | String | JWT Token |
| data.userId | Long | 用户ID |
| data.username | String | 用户名 |
| data.realName | String | 真实姓名 |
| data.roleIds | List\<Long\> | 角色ID列表 |
| data.permissions | List\<String\> | 权限标识列表 |
| data.menuTree | List\<MenuVO\> | 授权菜单树（目录+菜单，不含按钮） |

**业务规则**：
- 用户名或密码错误返回错误码 401
- 用户被停用时不允许登录

## 2. 用户管理

### 2.1 创建用户

**接口路径**：`POST /system/user/create`  
**权限标识**：`system:user:create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 用户名，最长50字符 |
| password | String | 是 | 密码，6-50字符 |
| realName | String | 否 | 真实姓名，最长50字符 |
| phone | String | 否 | 手机号，最长20字符 |
| email | String | 否 | 邮箱，最长100字符 |

**响应结构**：`R<UserVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 用户ID |
| data.username | String | 用户名 |
| data.realName | String | 真实姓名 |
| data.phone | String | 手机号 |
| data.email | String | 邮箱 |
| data.status | String | 状态：ACTIVE/INACTIVE |
| data.roleIds | List\<Long\> | 角色ID列表 |
| data.createdAt | LocalDateTime | 创建时间 |
| data.updatedAt | LocalDateTime | 更新时间 |

**业务规则**：
- 用户名必须唯一，重复时返回错误码 20001
- 创建后默认状态为 ACTIVE
- 密码使用 BCrypt 加密存储
- 当前密码策略：最少6位（PRD 要求：至少8位且含大小写和数字，90天过期，待后续迭代实现）

### 2.2 更新用户

**接口路径**：`POST /system/user/update`  
**权限标识**：`system:user:update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Long | 是 | Query参数，用户ID |
| realName | String | 否 | 真实姓名，最长50字符 |
| phone | String | 否 | 手机号，最长20字符 |
| email | String | 否 | 邮箱，最长100字符 |
| status | String | 否 | 状态：ACTIVE/INACTIVE |

**响应结构**：`R<UserVO>`

**业务规则**：
- 所有字段可选，只更新传入的字段
- 用户名不可修改

### 2.3 停用用户

**接口路径**：`POST /system/user/deactivate`  
**权限标识**：`system:user:deactivate`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Long | 是 | Query参数，用户ID |

**响应结构**：`R<Void>`

**业务规则**：
- 已停用的用户再次停用返回错误
- 停用后该用户无法登录

### 2.4 分页查询用户

**接口路径**：`POST /system/user/page`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | long | 否 | 页码，默认1 |
| size | long | 否 | 每页大小，默认10 |
| keyword | String | 否 | 关键词（用户名/姓名/手机号） |
| status | String | 否 | 状态筛选 |

**响应结构**：`R<IPage<UserVO>>`

分页结构见 [全局错误码与通用结构说明](../全局错误码说明.md)，`records` 为 `UserVO` 列表。

### 2.5 查询用户详情

**接口路径**：`POST /system/user/detail`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Long | 是 | Query参数，用户ID |

**响应结构**：`R<UserVO>`

### 2.6 分配角色

**接口路径**：`POST /system/user/assignRoles`  
**权限标识**：`system:user:assignRole`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | Long | 是 | Query参数，用户ID |
| roleIds | List\<Long\> | 是 | 角色ID列表，不能为空 |

**响应结构**：`R<Void>`

**业务规则**：
- 传入的 roleIds 会完全替换用户原有角色，不是追加

## 4. 菜单管理

### 4.1 创建菜单

**接口路径**：`POST /system/menu/create`  
**权限标识**：`system:menu:create`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| parentId | Long | 是 | 父菜单ID，0表示顶级 |
| name | String | 是 | 菜单名称 |
| type | String | 是 | 菜单类型：DIRECTORY/MENU/BUTTON |
| path | String | 否 | 路由路径 |
| component | String | 否 | 前端组件路径 |
| permission | String | 否 | 权限标识（按钮类型必填） |
| icon | String | 否 | 菜单图标 |
| sortOrder | Integer | 否 | 排序号 |
| visible | Boolean | 否 | 是否可见 |

**响应结构**：`R<MenuVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.id | Long | 菜单ID |
| data.parentId | Long | 父菜单ID |
| data.name | String | 菜单名称 |
| data.type | String | 类型：DIRECTORY/MENU/BUTTON |
| data.path | String | 路由路径 |
| data.component | String | 前端组件路径 |
| data.permission | String | 权限标识 |
| data.icon | String | 图标 |
| data.sortOrder | Integer | 排序号 |
| data.visible | Boolean | 是否可见 |
| data.status | String | 状态 |
| data.children | List\<MenuVO\> | 子菜单列表 |

### 4.2 更新菜单

**接口路径**：`POST /system/menu/update`  
**权限标识**：`system:menu:update`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| menuId | Long | 是 | Query参数，菜单ID |
| parentId | Long | 否 | 父菜单ID |
| name | String | 否 | 菜单名称 |
| type | String | 否 | 菜单类型 |
| path | String | 否 | 路由路径 |
| component | String | 否 | 前端组件路径 |
| permission | String | 否 | 权限标识 |
| icon | String | 否 | 图标 |
| sortOrder | Integer | 否 | 排序号 |
| visible | Boolean | 否 | 是否可见 |
| status | String | 否 | 状态：ACTIVE/INACTIVE |

**响应结构**：`R<MenuVO>`

### 4.3 删除菜单

**接口路径**：`POST /system/menu/delete`  
**权限标识**：`system:menu:delete`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| menuId | Long | 是 | Query参数，菜单ID |

**响应结构**：`R<Void>`

**业务规则**：
- 存在子菜单时不可删除

### 4.4 查询菜单树

**接口路径**：`POST /system/menu/tree`

**请求参数**：无

**响应结构**：`R<List<MenuVO>>`

返回完整的菜单树形结构，每个节点包含 `children` 子列表。

### 4.5 为角色分配菜单权限

**接口路径**：`POST /system/menu/assign`  
**权限标识**：`system:role:assignPermission`

**请求参数**（JSON Body）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roleId | Long | 是 | 角色ID |
| menuIds | List\<Long\> | 是 | 菜单ID列表（包含目录、菜单和按钮） |

**响应结构**：`R<Void>`

**业务规则**：
- 传入的 menuIds 完全替换角色原有权限

### 4.6 查询角色已分配的菜单ID列表

**接口路径**：`POST /system/menu/roleMenuIds`

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roleId | Long | 是 | Query参数，角色ID |

**响应结构**：`R<List<Long>>`

### 4.7 查询当前用户权限信息

**接口路径**：`POST /system/menu/permissions`

**请求参数**：无（从 JWT Token 中获取当前用户）

**响应结构**：`R<UserPermissionVO>`

| 字段 | 类型 | 说明 |
|------|------|------|
| data.menuTree | List\<MenuVO\> | 当前用户有权限的菜单树 |
| data.permissions | List\<String\> | 按钮级权限标识列表 |

## 5. 待实现功能

以下功能在 PRD 中已定义，但尚未开发，接口设计待实现后补充。

### 5.1 数据权限管理（US-SYS-003）

按角色配置数据可见范围（如只能看本部门/指定仓库的数据），与角色管理关联。

### 5.2 操作日志查询（US-SYS-004）

记录用户的关键操作（登录、新增、修改、删除等），支持按用户/时间/操作类型分页查询。

### 5.3 系统配置管理（US-SYS-005）

管理系统级参数（如密码过期天数、预留过期天数等键值对配置），支持 CRUD 和按分组查询。

### 5.4 审批规则配置（US-SYS-006）

配置审批流的规则（审批人、条件、级联等），与审批引擎模块（approval）联动，详见 `api/approval/` 目录。
