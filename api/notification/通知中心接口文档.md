# 通知中心接口文档

> 模块：notification（通知中心）
> 版本：v1.0
> 日期：2026-05-05
> 前置依赖：system 模块（用户/角色）、approval 模块（审批事件触发通知）

---

## 一、概述

通知中心提供站内消息管理和外部渠道推送能力。通知由业务事件触发（如审批结果、库存预警、订单状态变更），不对外暴露发送接口。

核心功能：
- 分页查询我的通知（含已读状态）
- 获取未读通知数量
- 标记已读 / 全部已读
- 查询/更新通知偏好（站内、企微、钉钉三个渠道开关）

通知分类（category）：
| 编码 | 说明 |
|------|------|
| APPROVAL | 审批通知 |
| INVENTORY_ALERT | 库存预警 |
| ORDER | 订单通知 |
| QUALITY | 质检通知 |
| STOCKTAKING | 盘点通知 |
| RETURN | 退货通知 |

## 二、通知查询

### 2.1 分页查询我的通知

**接口路径：** `POST /notification/page`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| category | String | 否 | 通知分类筛选（如 APPROVAL、ORDER） |
| isRead | Boolean | 否 | 已读状态筛选：true=已读, false=未读 |
| pageNum | Integer | 否 | 页码，默认 1 |
| pageSize | Integer | 否 | 每页条数，默认 20 |

**响应结构：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "current": 1,
    "size": 20,
    "total": 50,
    "records": [
      {
        "id": 1,
        "title": "销售订单审批通过",
        "content": "您的订单SO-20260501已通过审批",
        "category": "ORDER",
        "categoryLabel": "订单通知",
        "businessType": "SALES_ORDER",
        "businessId": 100,
        "businessNo": "SO-20260501",
        "senderId": null,
        "isRead": false,
        "readAt": null,
        "createdAt": "2026-05-05T10:30:00"
      }
    ]
  }
}
```

**业务规则：**
- 自动按当前登录用户过滤，只能看到发给自己的通知
- category 和 isRead 均为可选筛选条件
- 按创建时间倒序排列

### 2.2 获取未读通知数量

**接口路径：** `POST /notification/unread-count`

**请求参数：** 无

**响应结构：**
```json
{
  "code": 0,
  "message": "success",
  "data": { "unreadCount": 7 }
}
```

## 三、已读标记

### 3.1 标记单条通知已读

**接口路径：** `POST /notification/mark-read`
**权限标识：** `notification:read`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| notificationId | Long | 是 | 通知ID |

**响应结构：**
```json
{ "code": 0, "message": "success", "data": null }
```

**业务规则：**
- 如果通知已读或不存在，接口正常返回（幂等）
- 标记后记录阅读时间

### 3.2 标记所有通知已读

**接口路径：** `POST /notification/mark-all-read`
**权限标识：** `notification:readAll`

**请求参数：** 无

**响应结构：**
```json
{ "code": 0, "message": "success", "data": null }
```

**业务规则：**
- 将当前用户所有未读通知标记为已读
- 返回标记数量（通过响应 data 隐含，前端可重新查询未读数）

## 四、通知偏好

### 4.1 查询通知偏好列表

**接口路径：** `POST /notification/preference/detail`

**请求参数：** 无

**响应结构：**
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 1,
      "category": "APPROVAL",
      "categoryLabel": "审批通知",
      "channelSite": true,
      "channelWechat": true,
      "channelDingtalk": false
    },
    {
      "id": null,
      "category": "ORDER",
      "categoryLabel": "订单通知",
      "channelSite": true,
      "channelWechat": true,
      "channelDingtalk": false
    }
  ]
}
```

**业务规则：**
- 返回所有通知分类的偏好配置
- 如果用户尚未配置任何偏好，返回默认值（站内=开, 企微=开, 钉钉=关）
- id 为 null 表示该分类使用默认配置

### 4.2 更新通知偏好

**接口路径：** `POST /notification/preference/update`
**权限标识：** `notification:pref:update`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| category | String | 是 | 通知分类（如 APPROVAL、ORDER） |
| channelSite | Boolean | 否 | 站内通知开关，默认 true |
| channelWechat | Boolean | 否 | 企微推送开关，默认 true |
| channelDingtalk | Boolean | 否 | 钉钉推送开关，默认 false |

**响应结构：**
```json
{ "code": 0, "message": "success", "data": null }
```

**业务规则：**
- 按 userId + category 唯一，存在则更新，不存在则插入
- channel 字段为 null 时使用默认值

## 五、通知发送（内部接口，不对外暴露）

通知由业务事件触发，通过 `NotificationDomainService.sendNotification()` 调用。典型触发场景：

| 业务场景 | 通知分类 | 触发时机 |
|---------|----------|---------|
| 审批结果 | APPROVAL | 审批通过/驳回时 |
| 库存预警 | INVENTORY_ALERT | 库存低于安全库存时 |
| 订单状态变更 | ORDER | 订单确认/发货等状态变更时 |
| 质检结果 | QUALITY | 来料检验/成品检验完成时 |
| 盘点差异 | STOCKTAKING | 盘点差异超过阈值时 |
| 退货处理 | RETURN | 退货单状态变更时 |

**发送流程：**
1. 创建站内消息记录（t_sys_notification）
2. 为每个接收人创建接收记录（t_sys_notification_receiver）
3. 根据接收人通知偏好创建外部渠道推送记录（t_sys_notification_channel）
4. 调用外部 Webhook 推送（失败不回滚事务，仅记录失败原因）

**外部渠道推送配置（系统配置表 t_sys_config）：**
- `notification.wechat-work.webhook-url`：企微 Webhook 地址
- `notification.dingtalk.webhook-url`：钉钉 Webhook 地址

## 六、错误码

| 错误码 | 说明 |
|--------|------|
| 80001 | 通知推送失败 |
| 80002 | 通知不存在 |
| 80003 | 通知分类无效 |
| 80004 | 通知偏好更新失败 |
