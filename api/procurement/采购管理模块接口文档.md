# 采购管理模块接口文档

## 一、BOM管理

### 1.1 创建BOM

- **接口路径：** `POST /procurement/bom/create`
- **权限标识：** `procurement:bom:create`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| spuId | Long | 是 | 款式ID |
| name | String | 是 | BOM名称 |
| description | String | 否 | 描述 |
| items | List | 是 | BOM行列表 |

items 元素：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| materialId | Long | 是 | 物料ID |
| consumptionType | String | 是 | 消耗类型：FIXED_PER_PIECE/SIZE_DEPENDENT/PER_ORDER |
| baseConsumption | BigDecimal | 是 | 基准用量 |
| unit | String | 是 | 单位 |
| wastageRate | BigDecimal | 否 | 损耗率（如0.05表示5%） |
| sizeConsumptions | Map | 否 | 尺码用量表（SIZE_DEPENDENT时必填，key=sizeId，value=用量） |

**响应：** `R<BomVO>`

---

### 1.2 更新BOM

- **接口路径：** `POST /procurement/bom/update`
- **权限标识：** `procurement:bom:update`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bomId | Long | 是 | BOM ID |
| name | String | 否 | BOM名称 |
| description | String | 否 | 描述 |
| items | List | 是 | BOM行列表（全量替换） |

**响应：** `R<BomVO>`

---

### 1.3 删除BOM

- **接口路径：** `POST /procurement/bom/delete`
- **权限标识：** `procurement:bom:delete`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bomId | Long | 是 | BOM ID |

**响应：** `R<Void>`

**业务规则：** 已被生产订单引用的BOM不可删除。

---

### 1.4 审批生效BOM

- **接口路径：** `POST /procurement/bom/approve`
- **权限标识：** `procurement:bom:approve`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bomId | Long | 是 | BOM ID |

**响应：** `R<Void>`

**业务规则：** 同一SPU已有的APPROVED版本自动变为OBSOLETE。

---

### 1.5 查询BOM详情

- **接口路径：** `POST /procurement/bom/detail`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bomId | Long | 是 | BOM ID |

**响应：** `R<BomVO>`

---

### 1.6 分页查询BOM

- **接口路径：** `POST /procurement/bom/page`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| current | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页大小，默认20 |
| spuId | Long | 否 | 按款式ID过滤 |
| status | String | 否 | 按状态过滤：DRAFT/APPROVED/OBSOLETE |

**响应：** `R<IPage<BomVO>>`

---

## 二、MRP计算引擎

### 2.1 执行MRP计算

- **接口路径：** `POST /procurement/mrp/calculate`
- **权限标识：** `procurement:mrp:calculate`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| productionOrderIds | List | 否 | 指定生产订单ID列表（为空则计算所有待采购的生产订单） |

**响应：** `R<MrpCalculateResultVO>`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| batchNo | String | MRP计算批次号 |
| totalItems | Integer | 计算结果条数 |

**业务规则：**
- MRP六步流程：需求收集→BOM展开→物料合并→库存抵扣→采购建议→供应商匹配
- 库存查询使用预留接口（返回0，待库存模块实现后替换）
- 计算结果为"建议"，需人工审核后生成采购订单

---

### 2.2 查询MRP计算结果

- **接口路径：** `POST /procurement/mrp/results`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| current | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页大小，默认20 |
| batchNo | String | 否 | 按批次号过滤 |
| materialId | Long | 否 | 按物料ID过滤 |
| status | String | 否 | 按状态过滤：PENDING/APPROVED/CONVERTED/IGNORED/EXPIRED |

**响应：** `R<IPage<MrpResultVO>>`

---

## 三、采购订单管理

### 3.1 创建采购订单

- **接口路径：** `POST /procurement/order/create`
- **权限标识：** `procurement:order:create`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| supplierId | Long | 是 | 供应商ID |
| orderDate | String | 否 | 订单日期（yyyy-MM-dd，默认今天） |
| expectedDeliveryDate | String | 否 | 要求交货日期 |
| remark | String | 否 | 备注 |
| lines | List | 是 | 采购订单行列表 |

lines 元素：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| materialId | Long | 是 | 物料ID |
| materialType | String | 否 | 物料类型 |
| quantity | BigDecimal | 是 | 采购数量 |
| unit | String | 是 | 单位 |
| unitPrice | BigDecimal | 是 | 单价 |
| mrpResultId | Long | 否 | 来源MRP结果ID |
| remark | String | 否 | 备注 |

**响应：** `R<ProcurementOrderVO>`

**业务规则：**
- 创建后状态为DRAFT
- 行金额 = quantity × unit_price，总金额自动汇总
- 订单编号自动生成：PO-年月-5位流水号

---

### 3.3 分页查询采购订单

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| current | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页大小，默认20 |
| supplierId | Long | 否 | 按供应商过滤 |
| status | String | 否 | 按状态过滤 |

**响应：** `R<IPage<ProcurementOrderVO>>`

---

### 3.4 触发采购订单状态转移

- **接口路径：** `POST /procurement/order/fire-event`
- **权限标识：** `procurement:order:fire-event`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| orderId | Long | 是 | 采购订单ID |
| event | String | 是 | 事件名：SUBMIT/APPROVE/REJECT/RESUBMIT/ISSUE/RECEIVE/COMPLETE |

**响应：** `R<Void>`

**状态流转图：**
```
DRAFT → PENDING_APPROVAL (SUBMIT)
PENDING_APPROVAL → APPROVED (APPROVE)
PENDING_APPROVAL → REJECTED (REJECT)
REJECTED → PENDING_APPROVAL (RESUBMIT)
APPROVED → ISSUED (ISSUE)
ISSUED → RECEIVING (RECEIVE)
ISSUED → COMPLETED (COMPLETE)
RECEIVING → COMPLETED (COMPLETE)
```

---

**响应：** `R<List<String>>` — 可用事件名列表

## 四、到货通知单（ASN）

### 4.1 创建到货通知单

- **接口路径：** `POST /procurement/asn/create`
- **权限标识：** `procurement:asn:create`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| procurementOrderId | Long | 是 | 采购订单ID |
| supplierId | Long | 是 | 供应商ID |
| expectedArrivalDate | String | 否 | 预计到货日期（yyyy-MM-dd） |
| remark | String | 否 | 备注 |
| lines | List | 是 | 到货行列表 |

lines 元素：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| procurementLineId | Long | 是 | 采购订单行ID |
| materialId | Long | 是 | 物料ID |
| expectedQuantity | BigDecimal | 是 | 预计到货数量 |
| batchNo | String | 否 | 批次号 |
| remark | String | 否 | 备注 |

**响应：** `R<AsnVO>`

**业务规则：**
- 创建后状态为PENDING
- 到货通知数量不能超过采购订单行剩余可收数量
- ASN编号自动生成：ASN-年月-5位流水号

---

### 4.2 确认收货

- **接口路径：** `POST /procurement/asn/receive`
- **权限标识：** `procurement:asn:receive`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| asnId | Long | 是 | 到货通知单ID |
| lines | List | 是 | 各行实收信息 |

lines 元素：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| lineId | Long | 是 | 到货行ID |
| receivedQuantity | BigDecimal | 是 | 实收数量 |

**响应：** `R<Void>`

**业务规则：**
- PENDING或PARTIAL_RECEIVED状态才可收货
- 收货后更新采购订单行的已到货数量（delivered_quantity）
- 触发库存变更：在途库存减少，质检库存增加（预留接口）
- 记录实际到货日期和收货人

---

### 4.3 提交检验结果

- **接口路径：** `POST /procurement/asn/qc`
- **权限标识：** `procurement:asn:qc`

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| lineId | Long | 是 | 到货行ID |
| acceptedQuantity | BigDecimal | 是 | 合格数量 |
| rejectedQuantity | BigDecimal | 是 | 不合格数量 |
| inspector | String | 否 | 检验人 |
| conclusion | String | 否 | 结论说明 |
| items | List | 否 | 检验项明细 |

items 元素：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | String | 否 | 检验项名称（如：色差、克重、门幅） |
| standard | String | 否 | 标准要求 |
| actual | String | 否 | 实际值 |
| result | String | 否 | 检验结果：PASS/FAIL |

**响应：** `R<Void>`

**业务规则：**
- 合格数量 + 不合格数量 必须等于 实收数量
- 只有PENDING状态的行可提交检验结果
- 检验状态自动判定：
  - 不合格=0 → PASSED（合格）
  - 合格=0 → FAILED（不合格）
  - 两者都>0 → CONCESSION（让步接收）
- PASSED/CONCESSION → 质检库存转可用库存
- FAILED/CONCESSION的不合格部分 → 质检库存出库（退货）
- 更新采购订单行的accepted_quantity和rejected_quantity
- 检验结果完整保存到QcResult JSONB字段

---

### 4.4 查询到货通知单详情

**响应：** `R<AsnVO>`

---

### 4.5 分页查询到货通知单
