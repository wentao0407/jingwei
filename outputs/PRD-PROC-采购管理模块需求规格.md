# PRD-PROC 采购管理模块需求规格

> 模块编号：PROC  
> 优先级：P0（第二期交付）  
> 前置依赖：基础数据模块（物料、供应商、编码规则）、订单管理（生产订单、BOM）

---

## 一、模块概述

采购管理模块解决"生产需要什么原料、要买多少、找谁买、买回来质量合不合格"的问题。核心是MRP计算引擎，根据生产订单+BOM自动计算采购需求。

---

## 二、用户故事

### US-PROC-001 BOM管理

**作为** 技术员/计划员  
**我希望** 维护款式的物料清单(BOM)  
**以便** MRP计算各原料的采购需求量

**验收标准：**

1. 为每个款式创建BOM，添加面料/辅料/包材行项目
2. 面料行设为"尺码相关"消耗类型，录入每个尺码的用量（米/件）
3. 辅料行设为"每件固定"消耗类型，录入单件用量
4. 包材行设为"每件固定"或"按订单"消耗类型
5. 面料行设置损耗率（如8%），MRP计算时自动加损耗
6. BOM支持版本控制，同一款式可有多版本，仅一个"已审批"版本生效
7. 创建新版本时旧版本自动标记"已淘汰"
8. 生产订单关联具体BOM版本ID，不受后续版本变更影响

**边界场景：**

- 删除已关联生产订单的BOM → 不允许，提示"该BOM已被N张生产订单引用"
- 尺码用量表未填全所有尺码 → 提示"以下尺码未填写用量：XL、XXL"
- 损耗率输入超过50% → 警告"损耗率偏高，请确认"

---

### US-PROC-002 MRP计算

**作为** 计划员  
**我希望** 执行MRP计算，自动得出采购建议  
**以便** 知道需要采购什么、采购多少

**验收标准：**

1. 选择一张或多张生产订单，点击"计算采购需求"
2. 系统执行六步计算：需求汇总→BOM展开→同物料合并→库存抵扣→采购建议→供应商匹配
3. 展示计算结果列表：物料名称、毛需求、可用库存、在途数量、净需求、建议采购量
4. 每个物料的计算结果可展开查看明细（各生产订单各颜色的需求来源）
5. 面料计算考虑尺码用量差异和损耗率
6. 采购建议考虑供应商起订量(MOQ)和采购倍数
7. 计算过程耗时超过3秒时显示进度条

**边界场景：**

- 生产订单无BOM → 提示"以下款式无BOM，无法计算：XXX"
- 所有物料库存充足 → 提示"当前无需采购"，结果列表为空
- 重复计算同一生产订单 → 生成新批次，旧的未处理结果标记"已过期"

---

### US-PROC-003 采购订单创建

**作为** 采购员  
**我希望** 从MRP结果生成采购订单  
**以便** 向供应商下单采购

**验收标准：**

1. 勾选MRP结果行，点击"生成采购订单"
2. 系统按供应商自动分组，每个供应商生成一张采购订单
3. 自动填充：物料、数量、建议供应商、最近采购价
4. 采购员可调整数量、单价、交货日期
5. 采购订单行关联MRP结果行（来源可追溯）
6. 也可手动创建采购订单（不经过MRP）

**边界场景：**

- MRP结果行无推荐供应商 → 采购订单供应商为空，必须手动选择
- 修改采购数量低于起订量 → 警告"低于供应商起订量XXX"

---

### US-PROC-004 采购订单审批与下发

**作为** 采购经理  
**我希望** 审批采购订单后下发给供应商  
**以便** 控制采购成本和时机

**验收标准：**

1. 采购订单状态：草稿→待审批→已审批→已下发→到货中→已完成
2. 审批通过后可下发给供应商
3. 下发后系统记录在途库存
4. 审批驳回后采购员可修改重新提交
5. 下发后不可修改数量和价格

---

### US-PROC-005 到货通知与收货

**作为** 采购员/仓管员  
**我希望** 管理供应商到货通知和实际收货  
**以便** 跟踪到货进度并安排收货

**验收标准：**

1. 采购员创建到货通知单(ASN)，填写预计到货日期和各物料数量
2. 支持部分到货（一次到一部分，多次收货）
3. 不同次到货自动分配不同批次号
4. 收货时系统自动更新：在途库存减少，质检库存增加
5. 实收数量与预计数量有差异时记录差异原因

**边界场景：**

- 到货数量大于采购数量 → 允许，但标记"超量到货"供采购员确认
- 到货数量为0（全部没到）→ 允许记录，状态标记"未到货"

---

### US-PROC-006 来料检验

**作为** 质检员  
**我希望** 对到货物料进行来料检验  
**以便** 确保入库物料质量达标

**验收标准：**

1. 质检员在待检验列表中选择到货行，填写检验结果
2. 面料检验项：色差、克重、门幅、缩水率、色牢度、外观疵点
3. 辅料检验项：规格、颜色、数量、外观
4. 检验结论：合格/不合格/让步接收
5. 合格 → 质检库存减少，可用库存增加
6. 不合格 → 质检库存减少，生成退货出库单
7. 让步接收 → 标记降级，可用库存增加，记录偏差
8. 检验结果保存后不可修改（如需修改走调整流程）

---

### US-PROC-007 供应商对账

**作为** 采购员  
**我希望** 按期间与供应商对账  
**以便** 确认应付金额

**验收标准：**

1. 选择供应商和对账期间，系统自动汇总该期间的采购到货验收记录
2. 对账金额 = 检验合格数量 × 采购单价（不以采购数量或到货数量为准）
3. 对账单生成后可导出PDF给供应商确认
4. 供应商确认后对账单状态变为"已确认"
5. 对账存在争议时可标记"有异议"

## 三、业务规则

| 编号 | 规则 |
|------|------|
| BR-PROC-001 | 同款同色面料必须同一批次采购，不可拆分到不同采购订单 |
| BR-PROC-002 | BOM版本控制：同一SPU仅一个APPROVED版本 |
| BR-PROC-003 | MRP净需求 = 毛需求 - 可用库存 - 在途数量 |
| BR-PROC-004 | 采购建议量 = 净需求向上取整到采购倍数，不低于MOQ |
| BR-PROC-005 | 对账以检验合格数量为准，非采购/到货数量 |
| BR-PROC-006 | 不同次到货分配不同批次号 |
| BR-PROC-007 | 检验结果保存后不可修改 |
