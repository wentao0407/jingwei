1 系统整体架构 v2更新
经纬系统采用模块化单体（Modular Monolith）架构，物理上是一个部署单元，代码层面严格按限界上下文划分模块。模块之间通过领域事件+Outbox通信，不直接调用。

v2 变更点：① 移除 Spring Cloud Gateway，改用 Nginx 直连（<20人并发不需要网关层）；② 新增审批引擎、通知中心、退货管理、成本核算四个模块；③ 部署层移除 RabbitMQ 和 MinIO，初期不需要。

8 部署架构 v2更新
基于<20人并发规模，初期采用最简部署方案。后期业务增长时再演进为容器化微服务。

v2 变更点：① 移除Spring Cloud Gateway，Nginx直连Spring Boot；② 移除RabbitMQ，用Spring Event + Outbox；③ 移除MinIO，用本地文件系统；④ Redis仅用于Session/缓存/幂等令牌，不用于库存扣减。
